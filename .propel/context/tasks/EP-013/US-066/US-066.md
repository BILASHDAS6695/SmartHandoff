# US-066: Build SendGrid Dynamic Email Templates for Care Notifications

> **Epic:** EP-013 — Notification & Communication Infrastructure | **Sprint:** 2 | **Points:** 3 | **Priority:** Must Have
> **Status:** Draft | **Date:** 2026-07-14

---

## User Story

**As a** Backend Engineer,
**I want to** create SendGrid Dynamic Templates for all notification types (patient portal link, appointment reminder, medication reminder, care team escalation, ED boarding alert, housekeeping notification) and version-control them in source code,
**So that** every care-critical notification reaches patients and staff with consistent, properly formatted content regardless of which agent triggered it.

---

## Business Value

Consistent, well-formatted email templates improve patient trust and reduce confusion. A medication reminder email that looks professional and clearly states the medication name and schedule is significantly more likely to be acted upon than a plain-text generic notification. Templates also enable rapid content updates without code deployments.

---

## Acceptance Criteria

### Scenario 1: Patient portal link email renders correctly
**Given** a Twilio SendGrid notification is triggered for template `patient_portal_link`
**When** the email is rendered
**Then** the email shows: hospital logo, patient's first name greeting, portal link button (styled), discharge date, and footer with hospital contact information; the portal link is correct and unexpired.

### Scenario 2: All 6 template types render without errors in SendGrid Activity
**Given** all 6 templates are uploaded to SendGrid via CI/CD
**When** test sends are triggered for each template with sample substitution data
**Then** all 6 emails render without merge tag errors; the SendGrid Activity feed shows `delivered` status for each.

### Scenario 3: Templates are versioned in source control
**Given** a template is updated (e.g. appointment reminder wording changed)
**When** the CI/CD pipeline runs
**Then** the updated template JSON is uploaded to SendGrid via `SendGridAPIClient.client.templates.post()`; the previous version is archived in SendGrid; the change is tracked in Git history.

### Scenario 4: Medication reminder includes all required drug information
**Given** a medication reminder notification is triggered with `{drug_name: "Metformin", dose: "500mg", frequency: "twice daily", instructions: "with food"}`
**When** the email renders
**Then** the email shows: drug name, dose, frequency, special instructions, and a "Contact your care team if you have questions" section.

---

## Definition of Done

- [ ] 6 SendGrid Dynamic Template JSON files: `patient_portal_link`, `appointment_reminder`, `medication_reminder`, `care_team_escalation`, `ed_boarding_alert`, `housekeeping_notification`
- [ ] Templates stored in `notifications/templates/` in source control
- [ ] CI/CD step: `upload_sendgrid_templates.py` uploads templates to SendGrid API on every deploy
- [ ] Template substitution schemas: Pydantic models for each template's variables
- [ ] PHI minimisation: only `first_name` used in patient-facing templates; no MRN, DOB, or full name
- [ ] Unit tests: Pydantic schema validation for each template's substitution variables
- [ ] Code reviewed and approved

---

## Technical Notes

- SendGrid Dynamic Templates use Handlebars syntax: `{{first_name}}`, `{{portal_link}}`
- Template IDs stored in `config/sendgrid_templates.yaml` (updated by CI/CD upload script)
- Care team escalation and ED boarding: staff-facing templates, can include encounter ID (not PHI)
- Hospital logo: base64-encoded in template HTML or hosted on Cloud CDN
