# US-001: SMS and Email Notification Dispatch from Pub/Sub with Idempotency

> **Epic:** EP-013 — Notification & Communication Infrastructure | **Sprint:** 2 | **Points:** 5 | **Priority:** Must Have

## User Story

**As a** patient,
**I want** to receive SMS and email notifications reliably — such as my portal link, medication reminders, and appointment confirmations,
**so that** I'm always informed about my care plan and never miss a reminder due to a technical failure.

## Acceptance Criteria

- [ ] **AC-1:** Notification Service subscribes to `notification-requests` Pub/Sub topic; each message includes: `notification_id` (idempotency key), `channel` (SMS|EMAIL), `recipient` (phone or email), `template_name`, `template_data`, `send_at` (optional scheduled time)
- [ ] **AC-2:** Idempotency: `notification_id` checked in `notification_log` table before dispatch; duplicate `notification_id` → message ACK'd and skipped (no re-send); prevents duplicate messages from Pub/Sub redelivery
- [ ] **AC-3:** SMS dispatched via Twilio Programmable SMS API; delivery receipt tracked via webhook `POST /webhooks/twilio/status` → updates `notification_log.status` (SENT → DELIVERED or FAILED)
- [ ] **AC-4:** Email dispatched via SendGrid Dynamic Templates; all templates (portal link, appointment reminder, medication reminder, care team escalation, ED boarding, housekeeping) pre-configured and versioned in source control as JSON; deployed to SendGrid via CI
- [ ] **AC-5:** Failed delivery retried 3× with exponential backoff (1min, 2min, 4min); undeliverable after 3 retries → `notification_log.status=UNDELIVERABLE` → care team alert published to Pub/Sub
- [ ] **AC-6:** Scheduled notifications: `send_at` field defers dispatch until target time (Cloud Tasks or in-service scheduler); confirmed for 48-hour post-discharge check-ins
- [ ] **AC-7:** Patient opt-out: `patient.sms_opt_out=true` or `patient.email_opt_out=true` → notification skipped and logged as `OPTED_OUT`

## Definition of Done

- [ ] End-to-end SMS delivery confirmed in staging with real Twilio credentials
- [ ] Idempotency test: same `notification_id` sent twice → only one SMS delivered (Twilio logs confirm)
- [ ] Retry test: Twilio mock returns 500 × 3 → notification marked UNDELIVERABLE → alert fired
