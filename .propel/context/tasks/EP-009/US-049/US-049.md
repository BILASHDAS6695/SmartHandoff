# US-049: Build Patient List with Risk-Badge Display and Role-Based Filtering

> **Epic:** EP-009 — Care Team Dashboard & Real-Time Updates | **Sprint:** 2 | **Points:** 5 | **Priority:** Must Have
> **Status:** Draft | **Date:** 2026-07-14

---

## User Story

**As a** floor nurse (Nurse Nancy),
**I want to** see a searchable, filterable patient list showing only patients in my unit with colour-coded risk badges and quick-link to patient detail,
**So that** I can rapidly identify high-risk patients in my care area and prioritise my handoff tasks without sifting through the entire hospital census.

---

## Business Value

Role-based patient visibility (FR-074) is both a clinical workflow efficiency requirement and a HIPAA minimum-necessary access requirement. Nurses should only see patients in their unit — not the entire hospital — reducing cognitive overload and information security risk simultaneously.

---

## Acceptance Criteria

### Scenario 1: Nurse sees only own-unit patients
**Given** Nurse Nancy has `unit=3A` in her JWT claims
**When** she navigates to `/patients`
**Then** only encounters with `current_unit=3A` are displayed; patients from other units are NOT shown (server-enforced RBAC, not client-side filter).

### Scenario 2: Risk badges colour-coded by tier
**Given** the patient list loads
**When** encounters with different risk tiers are displayed
**Then** `HIGH` risk shows red badge, `MEDIUM` shows yellow badge, `LOW` shows green badge, and `UNSCORED` shows grey badge; all badges meet WCAG 2.1 AA contrast ratios.

### Scenario 3: Real-time risk badge updates via SignalR
**Given** a patient's risk score changes from MEDIUM to HIGH
**When** the `risk_score_updated` SignalR event is received
**Then** the risk badge in the patient list updates from yellow to red without page refresh within 1 second.

### Scenario 4: Patient list search and filter works correctly
**Given** the nurse searches for "Smith" in the patient list
**When** results are displayed
**Then** only patients with `last_name` containing "Smith" (from the nurse's unit) are shown; search is debounced 300ms to avoid API spam; skeleton loaders shown during loading.

## Definition of Done

- [ ] `PatientListComponent` with `MatTable` and virtual scrolling (CDK Virtual Scroll for >50 rows)
- [ ] Risk badge component: `<app-risk-badge [tier]="encounter.risk_tier">` — standalone reusable component
- [ ] RBAC: `GET /api/v1/patients?unit={unit}` enforces unit filter server-side; Angular reads unit from JWT claim
- [ ] Search: debounced 300ms `MatInput` with API query `?search={term}&unit={unit}`
- [ ] Unit filter dropdown: populated from nurse's assigned units (from JWT)
- [ ] Skeleton loaders (`MatSkeletonLoader`) during async load; error state with retry button
- [ ] `axe-core` test confirms WCAG 2.1 AA on this screen
- [ ] Code reviewed and approved

---

## Technical Notes

- JWT claim `units: ["3A"]` — multiple units possible for charge nurses
- Risk badge colours: CSS variables mapped to `--risk-high: #B71C1C`, `--risk-medium: #F57F17`, `--risk-low: #1B5E20`, `--risk-unknown: #616161`
- Patient list paginates at 25 per page with `MatPaginator`
