# US-003: Physician Document Approval Queue in Dashboard

> **Epic:** EP-009 — Care Team Dashboard & Real-Time Updates | **Sprint:** 2 | **Points:** 3 | **Priority:** Must Have

## User Story

**As an** attending physician,
**I want** a dedicated approval queue in the dashboard showing all discharge documents awaiting my review,
**so that** I can work through approvals efficiently without searching patient by patient.

## Acceptance Criteria

- [ ] **AC-1:** Physician dashboard shows an "Approvals" panel with badge count of pending documents; each entry shows: patient identifier (name visible to physician role), document type, generated time, encounter unit
- [ ] **AC-2:** Clicking an approval entry opens the Document Review screen (dual-pane editor from EP-004 US-004) in a side panel without navigating away from the queue
- [ ] **AC-3:** Approval queue sorted by: urgency (HIGH risk score patients first), then by document creation time ascending
- [ ] **AC-4:** Badge count on Approvals nav item updates in real time via SignalR `DocumentReadyForReview` event; new document ready = badge increments and toast notification fires
- [ ] **AC-5:** After physician approves a document: it disappears from approval queue within 2 seconds (optimistic UI update + confirmed by API response); badge count decrements

## Definition of Done

- [ ] Approval queue tested with 5 synthetic pending documents in staging
- [ ] SignalR badge update verified: approve document → badge decrements without page refresh
- [ ] Physician RBAC: approval queue not visible to NURSE role

## Dependencies

- EP-009 US-001 (Angular scaffold)
- EP-004 US-004 (Document review dual-pane editor)
- EP-003 US-003 (SignalR DocumentReadyForReview event)
- EP-011 (Physician RBAC)
