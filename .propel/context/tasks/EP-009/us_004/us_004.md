# US-004: Pharmacist Medication Reconciliation Queue with Priority Alerts

> **Epic:** EP-009 — Care Team Dashboard & Real-Time Updates | **Sprint:** 2 | **Points:** 3 | **Priority:** Must Have

## User Story

**As a** clinical pharmacist,
**I want** a priority-sorted medication reconciliation queue showing active alerts and pending reconciliations,
**so that** I can immediately triage the most dangerous drug interactions first.

## Acceptance Criteria

- [ ] **AC-1:** Pharmacist dashboard shows a Medication Queue panel: sorted by interaction severity (HIGH first), then by SLA elapsed time; each entry shows: patient ID (masked), medication pair or pending status, severity badge, time since admission
- [ ] **AC-2:** HIGH severity interaction alerts shown with red background and a "RESOLVE NOW" action button; clicking opens the Medication Review three-panel screen (EP-005 US-001) for that patient
- [ ] **AC-3:** Alert resolution logged via `PATCH /api/v1/alerts/{id}/resolve` with mandatory `resolution_type` (MEDICATION_CHANGED | ACCEPTED_WITH_MONITORING | ESCALATED) and `resolution_note`; alert removed from queue on resolution
- [ ] **AC-4:** SignalR `MedicationAlertCreated` event updates badge count on Medications nav item in real time; HIGH severity alert = browser notification sound (configurable, default on)
- [ ] **AC-5:** SLA countdown visible per patient: "X hours remaining before 24h SLA breach"; turns red at <2 hours remaining

## Definition of Done

- [ ] Medication queue tested with 3 HIGH + 2 MEDIUM alerts in staging; sort order verified
- [ ] Alert resolution removes item from queue within 2 seconds (optimistic UI)
- [ ] Pharmacist RBAC: queue not visible to NURSE or PHYSICIAN roles

## Dependencies

- EP-009 US-001 (Angular scaffold)
- EP-005 US-002 (Drug interaction alerts generated)
- EP-003 US-003 (SignalR MedicationAlertCreated event)
