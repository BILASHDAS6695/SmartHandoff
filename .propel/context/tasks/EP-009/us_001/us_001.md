# US-001: Angular PWA Scaffold with Role-Based Routing and SignalR Integration

> **Epic:** EP-009 — Care Team Dashboard & Real-Time Updates | **Sprint:** 2 | **Points:** 8 | **Priority:** Must Have

## User Story

**As a** frontend engineer,
**I want** the Angular 17 PWA scaffolded with lazy-loaded feature modules, in-memory JWT auth, and SignalR service,
**so that** the entire dashboard is built on a secure, performant foundation from day one.

## Acceptance Criteria

- [ ] **AC-1:** Angular 17 workspace with standalone components; feature modules lazy-loaded: `dashboard`, `patients`, `medications`, `documents`, `beds`, `analytics`, `patient-portal`, `admin`
- [ ] **AC-2:** `AuthGuard` redirects unauthenticated users to SSO login (OIDC); `JwtInterceptor` attaches Bearer token from in-memory store to all API calls — confirmed absent from `localStorage` and `sessionStorage` via browser DevTools test
- [ ] **AC-3:** `IdleTimeoutService` detects 30 minutes of user inactivity (no mouse/keyboard/API events) and triggers logout + SSO redirect
- [ ] **AC-4:** `SignalRService` establishes WebSocket connection to `/hubs/dashboard`; reconnects automatically on disconnect; handles `TaskStatusChanged` and `ADTEventReceived` event types
- [ ] **AC-5:** OpenAPI-generated TypeScript client from FastAPI OpenAPI spec (`openapi-generator`); API client regenerated in CI on spec change
- [ ] **AC-6:** Lighthouse CI audit in pipeline: LCP <2 seconds, main chunk <500KB — build fails if thresholds exceeded
- [ ] **AC-7:** axe-core accessibility CI check on all Must Have screens: zero WCAG 2.1 AA violations required

## Definition of Done

- [ ] `ng build --configuration=production` succeeds with main chunk <500KB verified
- [ ] SignalR reconnection tested by killing hub service and verifying auto-reconnect
- [ ] In-memory JWT confirmed absent from browser storage in Chrome DevTools session

## Dependencies

- EP-011 (OIDC SSO login endpoint and JWT)
- EP-003 US-003 (SignalR hub endpoint)
