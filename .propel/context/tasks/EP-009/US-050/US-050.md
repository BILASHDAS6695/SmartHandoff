# US-050: Render Visual Bed Board with Colour-Coded Status

> **Epic:** EP-009 — Care Team Dashboard & Real-Time Updates | **Sprint:** 2 | **Points:** 5 | **Priority:** Must Have
> **Status:** Draft | **Date:** 2026-07-14

---

## User Story

**As a** bed manager (Coordinator Carol),
**I want to** see a colour-coded visual bed board with all beds on the floor plan, their current status, and predicted discharge times, updated automatically within 60 seconds of each ADT event,
**So that** I can assess bed availability at a glance without making phone calls to nursing units.

---

## Business Value

The visual bed board is the primary operational tool for bed managers. A clear, self-updating floor-plan view reduces the average bed placement time from 30–45 minutes (manual phone calls) to <5 minutes (direct assignment from board), directly contributing to the 40% ED boarding reduction (BO-05).

---

## Acceptance Criteria

### Scenario 1: Bed board renders all beds with correct colour coding
**Given** the bed board page loads for a floor manager
**When** `GET /api/v1/beds` returns the current bed inventory
**Then** each bed cell is colour-coded: GREEN=VACANT, BLUE=OCCUPIED, ORANGE=DIRTY, GREY=MAINTENANCE; bed number, patient name (if occupied, masked to initials), and predicted discharge time are displayed in each cell.

### Scenario 2: Bed status updates within 60 seconds of ADT event
**Given** a nurse is viewing the bed board
**When** a SignalR `bed_status_changed` event is received
**Then** the affected bed cell updates colour and status text within 1 second of SignalR message receipt; no full page refresh required.

### Scenario 3: Clicking a bed cell opens patient detail
**Given** bed `3A-02` is OCCUPIED
**When** the bed manager clicks on the cell
**Then** a side panel slides open showing: patient name (if authorised), risk tier badge, predicted discharge time, and assigned nurse; a "Assign Bed" button for VACANT beds.

### Scenario 4: Bed board filters by unit
**Given** the hospital has units 3A, 3B, 4A, 4B, ICU
**When** the bed manager selects unit `ICU` in the unit filter
**Then** only ICU beds are displayed; other unit beds are hidden; the unit filter persists across navigation.

## Definition of Done

- [ ] `BedBoardComponent` with CSS Grid layout (responsive to 1024px–2560px viewports)
- [ ] Colour map: `{VACANT: green, OCCUPIED: blue, DIRTY: orange, MAINTENANCE: grey, RESERVED: purple}`
- [ ] Discharge prediction column (from `mv_bed_board.predicted_discharge_time`)
- [ ] SignalR handler: `bed_status_changed` event → update bed cell state
- [ ] Side panel: `BedDetailPanelComponent` with patient info (RBAC: show patient name only to authorised roles)
- [ ] Unit filter: `MatButtonToggle` group persisted to `sessionStorage`
- [ ] Skeleton loaders during initial load
- [ ] Responsive: 1024px (list view) to 2560px (full floor plan grid)
- [ ] Code reviewed and approved

---

## Technical Notes

- Floor plan layout: CSS Grid `repeat(auto-fill, minmax(120px, 1fr))` for responsive grid
- Patient name privacy: show initials (J.D.) for all roles; full name only for physician and charge_nurse roles
- `mv_bed_board` queried via `GET /api/v1/beds?include_predictions=true`
- WCAG: each bed cell needs `aria-label="Bed 3A-02, status Occupied, patient J.D., discharge predicted 3pm"`
