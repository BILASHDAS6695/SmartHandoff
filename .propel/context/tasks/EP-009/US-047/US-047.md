# US-047: Scaffold Angular 17 PWA Dashboard with Lazy-Loaded Modules

> **Epic:** EP-009 — Care Team Dashboard & Real-Time Updates | **Sprint:** 2 | **Points:** 8 | **Priority:** Must Have
> **Status:** Draft | **Date:** 2026-07-14

---

## User Story

**As a** frontend engineer,
**I want to** build the Angular 17 PWA foundation with workspace setup, lazy-loaded feature modules, Angular Material 17 with healthcare palette, CoreModule (auth guards, JWT interceptor, idle timeout), and the main dashboard layout,
**So that** all subsequent dashboard feature modules have a consistent, performant, and accessible base to build on.

---

## Business Value

The Angular scaffold is the prerequisite for all 5 dashboard feature stories. Without it, no frontend work can begin in parallel. The lazy-loading strategy is essential for the <2s load time SLA (TR-002) — eager loading all modules would triple the initial bundle size and fail the performance target.

---

## Acceptance Criteria

### Scenario 1: Dashboard loads within 2 seconds and main bundle <500KB
**Given** the Angular app is deployed to Cloud Run
**When** Lighthouse Performance audit runs on `4G` throttling
**Then** LCP (Largest Contentful Paint) < 2 seconds; main chunk size < 500KB (verified by webpack-bundle-analyzer).

### Scenario 2: Lazy-loaded modules do not load until navigation
**Given** a user lands on `/dashboard`
**When** the page loads
**Then** network inspector shows only the main chunk and dashboard module are loaded; `/patients`, `/beds`, `/analytics` modules are NOT in the initial load (confirmed by network tab).

### Scenario 3: Healthcare colour palette applied system-wide
**Given** the Angular Material theme is configured
**When** any component in the application is rendered
**Then** primary colour is `#0D47A1` (deep healthcare blue), accent is `#00897B` (teal), warn is `#B71C1C` (critical red); all WCAG 2.1 AA contrast ratios met.

### Scenario 4: JWT interceptor attaches Bearer token to all API requests
**Given** a user is authenticated and their JWT is stored in memory
**When** any HTTP request is made via `HttpClient`
**Then** the `Authorization: Bearer <token>` header is automatically attached by the `JwtInterceptor`; requests to non-API origins (CDN, external fonts) do NOT include the token.

## Technical Notes

- Use Angular standalone components (`@Component({standalone: true})`) for all new components
- `HttpClient` provideHttpClient with interceptors via `withInterceptors([jwtInterceptor])`
- Bundle budget: set `maximumWarning: "400kb"` and `maximumError: "500kb"` in `angular.json` build budget
- Dark mode: use CSS variables in `styles.scss` with Angular Material `ThemeService` toggle
