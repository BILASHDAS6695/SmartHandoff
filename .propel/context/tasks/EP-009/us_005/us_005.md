# US-005: Agent Activity Monitor for Supervisors

> **Epic:** EP-009 — Care Team Dashboard & Real-Time Updates | **Sprint:** 2 | **Points:** 3 | **Priority:** Should Have

## User Story

**As a** supervisor,
**I want** an Agent Monitor screen showing real-time performance metrics for all 6 AI agents,
**so that** I can identify failing agents, high queue depths, or SLA breaches before they affect patient care.

## Acceptance Criteria

- [ ] **AC-1:** Agent Monitor screen (`/admin/agents`) shows a table with one row per agent: agent name, tasks processed today, success rate (%), avg duration (ms), current queue depth (Pub/Sub undelivered message count), DLQ count
- [ ] **AC-2:** Failed tasks shown in a collapsible "Failed Tasks" section per agent: encounter_id, error message (non-PHI), retry count, timestamp; "Retry" button triggers `POST /api/v1/admin/agents/{type}/retry-failed`
- [ ] **AC-3:** System health panel shows: Cloud SQL replication lag, Pub/Sub subscription lag per agent, Cloud Run instance counts, Redis memory usage
- [ ] **AC-4:** Agent metrics refreshed every 30 seconds via polling (not SignalR); stale data indicator shown if last refresh >60 seconds ago
- [ ] **AC-5:** Screen accessible only to ADMIN role; hidden from all clinical roles

## Definition of Done

- [ ] Agent Monitor shows correct metrics for all 6 agents in staging
- [ ] Retry button successfully re-queues a failed test task
- [ ] ADMIN RBAC verified: other roles return 403 on `/admin/agents`

## Dependencies

- EP-009 US-001 (Angular scaffold)
- EP-003 US-001 (AgentTask records with status and error details)
- EP-011 (ADMIN RBAC)
