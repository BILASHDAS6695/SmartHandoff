# US-051: Build Medication Review Panel and Document Approval Queue

> **Epic:** EP-009 — Care Team Dashboard & Real-Time Updates | **Sprint:** 2 | **Points:** 5 | **Priority:** Must Have
> **Status:** Draft | **Date:** 2026-07-14

---

## User Story

**As a** clinical pharmacist (Pharmacist Phil),
**I want to** see a three-panel medication comparison view with interaction severity badges and alert resolution controls, and as a physician (Dr. David), I want a document approval queue showing AI-Assisted documents awaiting my review,
**So that** each role sees the exact workflow actions relevant to them without navigating to irrelevant screens.

---

## Business Value

Role-specific workflow panels eliminate the cognitive overload of a generic patient view. A pharmacist who only needs the medication panel and a physician who only needs the document queue are each more efficient in their specialised workflows, reducing per-patient review time and the risk of missing critical items.

---

## Acceptance Criteria

### Scenario 1: Pharmacist sees medication three-panel comparison with severity badges
**Given** Pharmacist Phil navigates to `/patients/{id}/medications`
**When** the reconciliation data loads
**Then** three columns display: "Pre-Admit", "Inpatient", "Discharge" with each drug row showing: drug name, dose, frequency, and a severity badge (RED=HIGH, YELLOW=MEDIUM, GREY=none) for detected interactions.

### Scenario 2: Alert resolution modal completes workflow
**Given** a HIGH-severity interaction alert badge is clicked
**When** the resolution modal opens
**Then** it shows: drug pair names, interaction description (from RxNav), severity, and resolution options (REVIEWED_ACCEPTABLE / DOSE_ADJUSTED / DRUG_CHANGED / DISCONTINUED); on submit, the badge clears and the alert status updates in real time.

### Scenario 3: Physician sees document approval queue on dashboard home
**Given** Dr. David navigates to `/dashboard`
**When** he has the `physician` role
**Then** an "Awaiting Approval" panel shows all `PENDING_REVIEW` documents for encounters in his assigned patients; count badge in sidebar navigation reflects the queue size.

### Scenario 4: Agent task status widget shows per-agent progress
**Given** an encounter has 5 agent tasks in various states
**When** the patient detail page loads
**Then** an "Agent Progress" card shows all 5 agents with status icons: ✓ COMPLETED, ⟳ IN_PROGRESS, ⏳ PENDING, ✗ FAILED; SLA breach is indicated with a red clock icon.

## Definition of Done

- [ ] `MedicationReviewComponent`: three-column `MatTable` with interaction severity badges
- [ ] `AlertResolutionModalComponent`: `MatDialog` with resolution type `MatRadioGroup` and note `MatTextarea`
- [ ] `DocumentQueueComponent`: `MatList` of `PENDING_REVIEW` documents with approve/reject quick actions
- [ ] `AgentProgressCardComponent`: reusable status icon card for all 5 agent types
- [ ] Role-based rendering: medication panel only for pharmacist/physician; document queue only for physician
- [ ] Toast on alert resolution: "Alert resolved — medication review complete"
- [ ] Error recovery: each panel has an error state with retry button (US-008 dependency on API errors)
- [ ] `axe-core` WCAG 2.1 AA test on both screens
- [ ] Code reviewed and approved

---

## Technical Notes

- Severity badge colours: match US-049 colour system — reuse `<app-risk-badge>` component with `severity` variant
- Document queue count badge: `MatBadge` on sidebar nav item, updated via SignalR `document_created` event
- Agent progress card uses `MatIcon` + custom status pipe: `status | agentStatusIcon` → icon name string
- Interaction description from API response: include truncated first 200 characters with "Read more" expansion
