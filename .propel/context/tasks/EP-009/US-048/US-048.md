# US-048: Integrate SignalR for Real-Time Dashboard Updates

> **Epic:** EP-009 — Care Team Dashboard & Real-Time Updates | **Sprint:** 2 | **Points:** 5 | **Priority:** Must Have
> **Status:** Draft | **Date:** 2026-07-14

---

## User Story

**As a** floor nurse on the care team dashboard,
**I want to** see ADT events, agent task completions, and alerts appear on my dashboard within 1 second without refreshing the page,
**So that** I can respond immediately to patient status changes rather than discovering them on my next manual refresh.

---

## Business Value

Real-time visibility transforms the dashboard from a passive display to an active operational tool. A nurse who sees an A03 discharge event appear in real time can immediately begin bed turnover coordination — reducing the typical 20-minute gap between discharge event and housekeeping notification.

---

## Acceptance Criteria

### Scenario 1: ADT event appears in live feed within 1 second
**Given** a nurse is viewing `/dashboard` with an active SignalR connection
**When** an A01 admission event is processed
**Then** a new entry appears in the "Live ADT Events" panel within 1 second with: event type, patient unit, timestamp, and encounter ID.

### Scenario 2: Agent task completion badge updates without page refresh
**Given** a nurse is viewing patient detail for encounter `ENC-001`
**When** the documentation agent task transitions to `COMPLETED`
**Then** the task status badge changes from "In Progress" to "Completed" within 1 second; a toast notification appears with the task name.

### Scenario 3: SignalR reconnects within 5 seconds of network interruption
**Given** the network connection drops for 3 seconds
**When** connectivity is restored
**Then** the SignalR service automatically reconnects within 5 seconds; a "Reconnected" toast appears; missed events are fetched via REST fallback poll.

### Scenario 4: Role-based group subscription filters events
**Given** a nurse in unit `3A` is connected to SignalR group `unit-3A`
**When** a `task_updated` event fires for an encounter in unit `4B`
**Then** the `3A` nurse does NOT receive the event; only nurses in unit `4B` or the encounter group `encounter-{id}` receive it.

---

## Definition of Done

- [ ] Angular `SignalRService`: `HubConnectionBuilder`, group subscriptions (`encounter-*`, `unit-*`, `role-*`), auto-reconnect with exponential backoff
- [ ] SignalR message handlers: `adt_event_received`, `task_updated`, `alert_created`, `bed_status_changed`
- [ ] Live ADT Events panel on `/dashboard`: real-time feed (last 20 events) with virtual scrolling
- [ ] Task status badge component: subscribes to `task_updated` events for displayed encounter
- [ ] REST fallback: on reconnect, poll `GET /api/v1/encounters/recent-events?since={last_event_time}`
- [ ] Toast notifications: `MatSnackBar` for agent task completion and high-priority alerts
- [ ] Integration test: server-to-client event latency ≤1 second (measured with performance.now())
- [ ] Code reviewed and approved

---

## Technical Notes

- `@microsoft/signalr` npm package (v7.x)
- Connection URL: `{API_BASE_URL}/hubs/dashboard?access_token={jwt}` — token in query param (SignalR limitation)
- Group subscription on connect: `hubConnection.invoke("JoinGroups", {units: ["3A"], roles: ["NURSE"]})`
- Reconnect strategy: `withAutomaticReconnect([0, 2000, 5000, 10000, 30000])` (ms delays)
