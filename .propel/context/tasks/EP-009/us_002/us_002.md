# US-002: Live ADT Event Feed and Patient Risk Dashboard

> **Epic:** EP-009 — Care Team Dashboard & Real-Time Updates | **Sprint:** 2 | **Points:** 5 | **Priority:** Must Have

## User Story

**As a** floor nurse,
**I want** a live ADT event feed and colour-coded patient risk list on my dashboard,
**so that** I always know who was admitted, transferred, or discharged — and can immediately identify high-risk patients who need extra attention.

## Acceptance Criteria

- [ ] **AC-1:** Dashboard Home (`/dashboard`) shows a live ADT event feed: last 50 events, auto-updated via SignalR `ADTEventReceived`; each entry shows: event type badge (ADMIT/TRANSFER/DISCHARGE), unit, masked MRN, timestamp, elapsed time
- [ ] **AC-2:** Patient List (`/patients`) shows risk-score badge per patient: green (<0.3), amber (0.3–0.7), red (>0.7); sortable by risk score descending; unit filter defaults to logged-in nurse's unit (RBAC-scoped)
- [ ] **AC-3:** Nurse sees only patients on their own unit (from `mv_risk_dashboard WHERE unit={nurse_unit}`); pharmacist sees all patients; physician sees their own patients — RBAC enforced at API level
- [ ] **AC-4:** New ADT event causes a toast notification (top-right, 5-second auto-dismiss) with event type and unit; badge count increments on Dashboard nav item
- [ ] **AC-5:** Patient List search: debounced (300ms) text search across patient name (server-side, encrypted-field search via MRN lookup) with minimum 3-character trigger
- [ ] **AC-6:** Skeleton loaders displayed during initial data fetch; error boundary shows "Unable to load patients — retry" with retry button if API fails

## Definition of Done

- [ ] Live ADT feed verified in staging: send A01 → feed updates within 1 second via SignalR
- [ ] RBAC test: nurse account sees only own-unit patients; API returns 200 with filtered data
- [ ] Lighthouse score ≥85 on Dashboard Home

## Dependencies

- EP-003 US-003 (SignalR events)
