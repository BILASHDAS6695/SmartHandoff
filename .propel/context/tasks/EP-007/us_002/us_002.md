# US-002: Activate High-Risk Care Pathway for Patients with Risk Score ≥0.7

> **Epic:** EP-007 — Follow-up Care Agent & Risk Scoring | **Sprint:** 2 | **Points:** 3 | **Priority:** Must Have

## User Story

**As a** care manager,
**I want** high-risk patients (risk score ≥0.7) automatically enrolled in an intensive follow-up pathway,
**so that** no high-risk patient leaves the hospital without a mandatory 7-day follow-up appointment and a care manager assigned.

## Acceptance Criteria

- [ ] **AC-1:** On risk score ≥0.7: Agent sets `Encounter.risk_tier=HIGH`, creates appointment record with `scheduled_within_days=7`, publishes care manager alert to `notification-requests`
- [ ] **AC-2:** Care manager receives SignalR dashboard alert AND SMS with: patient encounter ID, risk score, top risk factors, and a link to schedule follow-up
- [ ] **AC-3:** System enforces mandatory follow-up: `Encounter.followup_scheduled` must be `true` before encounter fully closes; if not scheduled within 7 days → SLA breach alert to clinical director
- [ ] **AC-4:** HIGH risk patients flagged with red banner in Patient List and Patient Detail views in Angular dashboard; banner reads "HIGH READMISSION RISK — Follow-up Required"
- [ ] **AC-5:** Risk score ≥0.5 (but <0.7): standard 48-hour check-in message scheduled (US-003); risk score <0.3: standard 30-day follow-up only

## Definition of Done

- [ ] High-risk pathway tested end-to-end: A03 with score ≥0.7 → care manager alert received → appointment record created → red banner visible in dashboard
- [ ] 7-day mandatory follow-up SLA enforced (integration test confirms alert after 8 days with no follow-up)

## Dependencies

- EP-007 US-001 (Risk score calculated)
- EP-013 US-001 (SMS notification for care manager)
- EP-003 US-003 (SignalR push for care manager dashboard alert)
