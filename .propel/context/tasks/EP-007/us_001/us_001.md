# US-001: Calculate 30-Day Readmission Risk Score at Discharge

> **Epic:** EP-007 — Follow-up Care Agent & Risk Scoring | **Sprint:** 2 | **Points:** 8 | **Priority:** Must Have

## User Story

**As a** care manager,
**I want** a 30-day readmission risk score calculated automatically at discharge,
**so that** I can identify high-risk patients who need intensive follow-up and prevent unnecessary readmissions.

## Acceptance Criteria

- [ ] **AC-1:** Follow-up Care Agent subscribes to `followup-sub`; on A03 (Discharge): calls ML Inference Service `POST /predict/readmission` with features: `{patient_age, primary_diagnosis_code, comorbidity_count, length_of_stay_days, prior_readmissions_30d, discharge_disposition}`
- [ ] **AC-2:** ML Inference Service returns `{risk_score: 0.0-1.0, risk_tier: LOW|MEDIUM|HIGH, top_risk_factors: [{factor, contribution}]}`; response within 500ms
- [ ] **AC-3:** Scikit-learn LogisticRegression model achieves AUC ≥0.80 on 20% holdout test set from historical encounter data
- [ ] **AC-4:** `Encounter.risk_score` and `Encounter.risk_tier` persisted to DB; `mv_risk_dashboard` materialised view updated within 5 minutes
- [ ] **AC-5:** Dashboard shows colour-coded risk badge per patient: green (<0.3), amber (0.3–0.7), red (>0.7)
- [ ] **AC-6:** Risk score and top contributing factors accessible via `GET /api/v1/encounters/{id}/risk`; top 3 risk factors listed in plain language (e.g., "Prior readmission in last 30 days")

## Definition of Done

- [ ] Model AUC ≥0.80 verified on holdout dataset
- [ ] Risk score visible in dashboard within 5 minutes of A03 event in staging
- [ ] p95 ML inference <500ms under 50 concurrent requests

## Dependencies

- EP-003 US-001 (AgentTask framework)
- EP-DATA US-001 (Encounter risk_score/risk_tier columns)
- EP-TECH US-001 (ML Inference Cloud Run service)
