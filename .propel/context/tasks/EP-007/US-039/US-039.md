# US-039: Calculate 30-Day Readmission Risk Score at Discharge

> **Epic:** EP-007 — Follow-up Care Agent & Risk Scoring | **Sprint:** 2 | **Points:** 5 | **Priority:** Must Have
> **Status:** Draft | **Date:** 2026-07-14

---

## User Story

**As a** care manager,
**I want to** receive a 30-day readmission risk score (0.0–1.0) for every discharged patient, calculated within 60 seconds of the A03 event, with a clear risk tier (LOW/MEDIUM/HIGH),
**So that** I can immediately prioritise post-discharge follow-up resources for patients most likely to return to the hospital within 30 days.

---

## Business Value

30-day readmissions cost US hospitals $26B+ annually. ML risk scoring identifies the 20% of patients who generate 80% of readmissions, enabling targeted intervention. A >80% AUC model (BO-09) provides enough signal to meaningfully differentiate care pathways, directly reducing readmission rate by 25% (BO-02).

---

## Acceptance Criteria

### Scenario 1: Risk score calculated within 60 seconds of A03 event
**Given** an A03 discharge event is processed for encounter `ENC-001`
**When** the follow-up care agent runs
**Then** `encounter.risk_score` and `encounter.risk_tier` are persisted within 60 seconds; `GET /api/v1/encounters/ENC-001/risk` returns the score.

### Scenario 2: Risk tier assignment is correct
**Given** model returns probability=0.25
**Then** `risk_tier=LOW`; for probability=0.55, `risk_tier=MEDIUM`; for probability=0.72, `risk_tier=HIGH`.

### Scenario 3: ML model AUC ≥0.80 on holdout test set
**Given** the LogisticRegression model trained on historical encounter data
**When** evaluated on the 20% holdout set
**Then** AUC-ROC ≥ 0.80; this is documented in the model evaluation report stored in GCS.

### Scenario 4: Risk score and contributing factors visible via API
**Given** a completed risk assessment for encounter `ENC-001`
**When** `GET /api/v1/encounters/ENC-001/risk` is called with a physician JWT
**Then** the response includes: `risk_score`, `risk_tier`, `contributing_factors` (top 5 feature importances as human-readable labels), and `model_version`.

## Definition of Done

- [ ] `FollowUpCareAgent` extends `BaseAgent`; triggered by A03 discharge events
- [ ] ML Inference endpoint: `POST /ml-inference/predict/readmission` with encounter features
- [ ] Features: `primary_diagnosis_group`, `age`, `los_days`, `num_comorbidities`, `num_prior_admissions_12mo`, `discharge_disposition`, `medication_count`
- [ ] Risk tier thresholds: `LOW: <0.3, MEDIUM: 0.3–0.7, HIGH: ≥0.7`
- [ ] `encounter.risk_score` (float) and `encounter.risk_tier` (enum) columns (add via Alembic migration)
- [ ] `GET /api/v1/encounters/{id}/risk` endpoint with `contributing_factors` computed via SHAP values
- [ ] Model evaluation report: AUC, precision, recall, F1 on holdout set
- [ ] Code reviewed and approved

---

## Technical Notes

- Scikit-learn `LogisticRegression` with L2 regularisation; feature scaling via `StandardScaler`
- SHAP `explainer.shap_values()` for contributing factors; map to human-readable labels in `config/feature_labels.yaml`
- Model stored in GCS `ml-models/readmission-risk/v{N}.joblib`
- Features requiring FHIR: `num_comorbidities` from `Condition` resource; `prior_admissions` from encounter history in SmartHandoff DB
