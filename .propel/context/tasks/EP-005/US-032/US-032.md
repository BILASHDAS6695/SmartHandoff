# US-032: Alert Pharmacist for High-Risk Drug Classes at Discharge

> **Epic:** EP-005 — Medication Reconciliation Agent | **Sprint:** 2 | **Points:** 3 | **Priority:** Must Have
> **Status:** Draft | **Date:** 2026-07-14

---

## User Story

**As a** clinical pharmacist,
**I want to** receive a mandatory pharmacist alert whenever a patient is discharged with a high-risk drug class (anticoagulants, insulin, opioids) — regardless of interaction severity — so that I can review these high-risk prescriptions before the patient leaves.

---

## Business Value

ISMP (Institute for Safe Medication Practices) designates anticoagulants, insulin, and opioids as "High-Alert Medications" due to their disproportionate harm when misused. Mandatory review of these drug classes at discharge prevents the most common categories of post-discharge adverse drug events, directly supporting BO-01 (medication error rate <10%).

---

## Acceptance Criteria

### Scenario 1: Anticoagulant on discharge list triggers mandatory alert
**Given** `Warfarin 5mg` (anticoagulant) is on a patient's discharge medication list
**When** the high-risk drug class detector runs
**Then** a `PHARMACIST_ALERT` is created with `alert_type=HIGH_RISK_DRUG_CLASS`, `drug_class=ANTICOAGULANT`, `drug_name=Warfarin`, `severity=HIGH` — regardless of whether any interaction was found.

### Scenario 2: Alert resolution workflow completed by pharmacist
**Given** a HIGH_RISK_DRUG_CLASS alert exists for encounter `ENC-001`
**When** Pharmacist Phil calls `PATCH /api/v1/alerts/{id}/resolve` with `resolution_type=REVIEWED_ACCEPTABLE` and a note
**Then** `Alert.status=RESOLVED`, `Alert.resolved_by_user_id` set to Phil's user ID, `Alert.resolved_at` set to current timestamp; the alert disappears from the active pharmacist queue.

### Scenario 3: 24-hour SLA breach escalates to charge pharmacist
**Given** a HIGH-severity pharmacist alert was created 24 hours ago and remains unresolved
**When** the SLA monitor runs
**Then** a `CHARGE_PHARMACIST_ESCALATION` notification is published to `notification-requests`; the alert is tagged `sla_breached=True`.

### Scenario 4: Alert resolution blocked for non-pharmacist role
**Given** a nurse attempts to call `PATCH /api/v1/alerts/{id}/resolve`
**When** the API is called with a nurse JWT
**Then** the response is `403 Forbidden`; the alert status remains unchanged.

## Definition of Done

- [ ] `HighRiskDrugClassDetector` class with configurable high-risk drug classes (YAML)
- [ ] High-risk classes: `ANTICOAGULANT`, `INSULIN`, `OPIOID`, `CHEMOTHERAPY` (extensible list)
- [ ] Drug-to-class mapping: RxNorm ATC classification or `config/high_risk_drugs.yaml` lookup table
- [ ] `POST /api/v1/encounters/{id}/alerts` endpoint for alert creation
- [ ] `PATCH /api/v1/alerts/{id}/resolve` endpoint with RBAC (pharmacist-only)
- [ ] Alert SLA monitor: 24h threshold for medication alerts
- [ ] Unit tests: each high-risk class, RBAC enforcement, SLA breach
- [ ] Code reviewed and approved

---

## Technical Notes

- High-risk drug YAML: `{ANTICOAGULANT: [warfarin, heparin, enoxaparin, rivaroxaban], INSULIN: [insulin glargine, insulin aspart, ...], OPIOID: [oxycodone, hydrocodone, morphine, ...]}` — case-insensitive name match against RxNorm preferred name
- Alert type is ADDITIVE: a drug can trigger both a drug-interaction alert AND a high-risk drug class alert simultaneously
- Resolution types: `REVIEWED_ACCEPTABLE`, `DOSE_ADJUSTED`, `DRUG_CHANGED`, `DISCONTINUED`
