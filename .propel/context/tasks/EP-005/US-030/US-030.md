# US-030: Compare Three FHIR Medication Lists and Categorise Changes

> **Epic:** EP-005 — Medication Reconciliation Agent | **Sprint:** 2 | **Points:** 5 | **Priority:** Must Have
> **Status:** Draft | **Date:** 2026-07-14

---

## User Story

**As a** clinical pharmacist (Pharmacist Phil),
**I want to** see a three-way comparison of a patient's pre-admission, inpatient, and discharge medication lists with each drug categorised as CONTINUED, NEW, STOPPED, or DOSE_CHANGED,
**So that** I can instantly identify all medication changes during the hospital stay without manually reviewing three separate medication lists.

---

## Business Value

Medication discrepancies at care transitions cause 66% of adverse drug events. Three-list FHIR reconciliation (FR-030) is the foundational analysis that surfaces all changes, enabling the pharmacist to catch errors before the patient leaves with incorrect medications — directly achieving BO-01 (medication error rate <10%).

---

## Acceptance Criteria

### Scenario 1: All three FHIR lists fetched and compared correctly
**Given** a patient with 5 pre-admission meds, 7 inpatient meds, and 4 discharge meds in FHIR
**When** the medication reconciliation agent runs
**Then** the comparison result categorises each medication: `CONTINUED` (in pre-admit AND discharge), `NEW` (only in discharge), `STOPPED` (in pre-admit, absent from discharge without documented stop order), `DOSE_CHANGED` (same drug, different dose or route).

### Scenario 2: Duplicate medication detected
**Given** a patient has `Metformin 500mg oral` and `Metformin XR 500mg oral` on the discharge list
**When** the duplicate detector runs
**Then** the pair is flagged as `DUPLICATE` with a warning; a pharmacist alert is created with severity `MEDIUM`.

### Scenario 3: Missing chronic medication detected
**Given** a patient was on `Atorvastatin 40mg` pre-admission but it is absent from the discharge list without a documented stop order
**When** the missing medication checker runs
**Then** `Atorvastatin 40mg` is flagged as `STOPPED_WITHOUT_ORDER` with severity `HIGH`; a pharmacist alert is created.

### Scenario 4: Reconciliation comparison persisted and returned via API
**Given** the reconciliation agent has completed its analysis
**When** `GET /api/v1/encounters/{id}/medications/reconciliation` is called
**Then** the response contains all drugs with their `reconciliation_category`, `pre_admit`, `inpatient`, and `discharge` list membership, and `flags` (DUPLICATE, STOPPED_WITHOUT_ORDER, etc.).

## Definition of Done

- [ ] `MedicationReconciliationAgent` extends `BaseAgent`
- [ ] FHIR fetch: `MedicationStatement` (pre-admit), `MedicationAdministration` (inpatient), `MedicationRequest` (discharge)
- [ ] Normalisation step: map drug names to RxNorm CUI via RxNav normalisation API
- [ ] Comparison algorithm: categorise all drugs as CONTINUED/NEW/STOPPED/DOSE_CHANGED
- [ ] Duplicate detection: same RxNorm CUI + same route → DUPLICATE flag
- [ ] Missing chronic detection: pre-admit drug absent from discharge without stop order → `STOPPED_WITHOUT_ORDER`
- [ ] Results persisted to `medication` ORM table with reconciliation fields
- [ ] `GET /api/v1/encounters/{id}/medications/reconciliation` FastAPI endpoint
- [ ] Unit tests with 15+ medication test fixtures covering all categories
- [ ] Code reviewed and approved

---

## Technical Notes

- RxNorm normalisation: `GET https://rxnav.nlm.nih.gov/REST/rxcui?name={drug_name}` for CUI lookup
- Dose comparison: parse dose string (e.g. "500mg") into `{value: 500, unit: "mg"}` for numerical comparison
- Stop order detection: check for a `FHIR MedicationRequest.status=stopped` record for the drug
- Medication ORM: store `rxnorm_cui`, `reconciliation_category`, `dose`, `route`, `frequency` for each drug
