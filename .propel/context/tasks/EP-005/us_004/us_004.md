# US-004: Generate Plain-Language Patient Medication Change Summary

> **Epic:** EP-005 — Medication Reconciliation Agent | **Sprint:** 2 | **Points:** 3 | **Priority:** Must Have

## User Story

**As a** patient,
**I want** a simple summary of what happened to my medications during my hospital stay,
**so that** I understand which pills to take, which to stop, and why — without needing to ask a pharmacist.

## Acceptance Criteria

- [ ] **AC-1:** Agent generates a patient-readable medication summary using Gemini with the categorised medication changes as input; prompt instructs: "Write in plain English, avoid medical jargon, explain *why* each medication changed"
- [ ] **AC-2:** Summary structured as: "Medications you should continue:", "New medications:", "Medications you should stop:", "Dose changes:"; each entry includes drug name (brand + generic), dose, frequency, and a one-sentence reason in plain language
- [ ] **AC-3:** Summary persisted as `Document` record with `document_type=MED_SUMMARY`; Flesch-Kincaid Grade ≤8 (slightly higher than discharge instructions, acceptable for medication context)
- [ ] **AC-4:** Summary automatically attached to patient portal under "Your Medications" section after physician approves discharge documentation
- [ ] **AC-5:** If patient has no medication changes (all CONTINUED): summary still generated stating "All your regular medications continue unchanged"

## Definition of Done

- [ ] 10 test patient scenarios verified: summaries accurate, readable, and clinically correct
- [ ] Patient portal displays medication summary in correct section (EP-010)

## Dependencies

- EP-005 US-001 (Medication categorisation complete)
- EP-004 US-001 (Discharge summary approved — prerequisite for portal activation)
