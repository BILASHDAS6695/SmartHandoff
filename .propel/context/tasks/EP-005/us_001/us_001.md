# US-001: Compare Three Medication Lists and Categorise All Changes

> **Epic:** EP-005 — Medication Reconciliation Agent | **Sprint:** 2 | **Points:** 8 | **Priority:** Must Have

## User Story

**As a** clinical pharmacist,
**I want** all three medication lists (pre-admission, inpatient, discharge) compared side by side with every change categorised,
**so that** I can quickly identify what changed during the hospital stay and verify the discharge prescription is correct.

## Acceptance Criteria

- [ ] **AC-1:** Agent fetches 3 FHIR medication lists: `MedicationStatement` (pre-admission), `MedicationAdministration` (inpatient), `MedicationRequest` (discharge prescription) via EP-002 FHIR client
- [ ] **AC-2:** Comparison algorithm matches medications by RxCUI (preferred) or drug name + route; categorises each as: CONTINUED (unchanged), NEW (added during stay), STOPPED (on pre-admit list, absent from discharge), DOSE_CHANGED (same drug, different dose/frequency/route)
- [ ] **AC-3:** Each `Medication` record persisted to DB with `list_type`, `status` (change category), `rxcui`, `drug_name`, `dosage`, `frequency`, `route`
- [ ] **AC-4:** Pharmacist Medication Review screen shows three-panel view: Pre-Admission | Inpatient | Discharge — with colour coding: green=CONTINUED, blue=NEW, red=STOPPED, amber=DOSE_CHANGED
- [ ] **AC-5:** Reconciliation must be initiated within 24 hours of A01 event (BR-002); if not started, SLA breach escalation fires (EP-003 US-002)
- [ ] **AC-6:** Integration test: synthetic patient with 8 pre-admission meds (3 continued, 2 new, 2 stopped, 1 dose-changed) → verify all 8 correctly categorised

## Definition of Done

- [ ] 95%+ categorisation accuracy verified on 20 synthetic medication reconciliation test cases
- [ ] Three-panel UI reviewed by a pharmacist in UAT

## Dependencies

- EP-002 US-001 (FHIR client for medication list fetching)
- EP-DATA US-001 (Medication ORM model)
- EP-003 US-001 (AgentTask framework)
