# US-004: Store All Chatbot Transcripts Encrypted Against Patient Encounter

> **Epic:** EP-008 — Patient Communication Agent & Chatbot | **Sprint:** 2 | **Points:** 2 | **Priority:** Must Have

## User Story

**As a** care team member,
**I want** all patient chatbot conversations stored against the encounter record,
**so that** I can review what concerns the patient raised after discharge and use it to improve post-discharge care.

## Acceptance Criteria

- [ ] **AC-1:** Every chatbot message (both patient and agent directions) persisted as `ChatbotTranscript` record: `encounter_id`, `direction` (PATIENT|AGENT), `message` (AES-256-GCM encrypted), `urgency_flag`, `escalated`, `sent_at`
- [ ] **AC-2:** Transcripts linked to encounter record; care team can view full conversation via `GET /api/v1/encounters/{id}/transcripts` (requires PHYSICIAN or NURSE role)
- [ ] **AC-3:** Transcript message content encrypted at rest (AES-256-GCM via EP-DATA US-002 TypeDecorator); confirmed via direct DB query showing ciphertext
- [ ] **AC-4:** Conversation history in Redis (`conversation-history:{encounter_id}`) cleared 30 days post-discharge; DB transcript retained for 7 years with encounter
- [ ] **AC-5:** `urgency_flag=true` correctly set on transcripts where urgency detection triggered (US-002); care team can filter transcripts by `urgency_flag=true` for safety review

## Definition of Done

- [ ] Transcript encryption verified via psql ciphertext check
- [ ] Urgency flag correctly populated for 10 urgency test messages
- [ ] RBAC verified: patient cannot access transcripts via API (only via portal chatbot widget); care team can read

## Dependencies

- EP-008 US-001 (Chatbot API established)
- EP-DATA US-002 (PHI encryption TypeDecorator)
- EP-011 (RBAC for transcript read access)
