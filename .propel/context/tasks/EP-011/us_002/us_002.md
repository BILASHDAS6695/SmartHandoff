# US-002: HIPAA-Compliant PHI Audit Logging on Every Access

> **Epic:** EP-011 — Security, Auth & HIPAA Compliance | **Sprint:** 1 | **Points:** 5 | **Priority:** Must Have

## User Story

**As a** compliance officer,
**I want** every access to PHI-containing endpoints logged immutably with user identity, action, and timestamp,
**so that** I can prove HIPAA compliance in any audit and investigate any suspected data breach.

## Acceptance Criteria

- [ ] **AC-1:** FastAPI middleware (`HIPAAAuditMiddleware`) runs on every request to PHI-bearing endpoints (`/api/v1/patients/*`, `/api/v1/encounters/*`, `/api/v1/documents/*`, `/api/v1/medications/*`, `/api/v1/chat/transcripts/*`); creates `AuditLog` entry with: `user_id`, `user_role`, `action` (READ|WRITE|APPROVE|EXPORT), `entity_type`, `entity_id`, `ip_address`, `user_agent`, `action_at`
- [ ] **AC-2:** PHI log sanitiser middleware runs on all logging calls: strips `first_name`, `last_name`, `dob`, `phone`, `email`, `mrn` from log payloads before emission to Cloud Logging; verified by searching Cloud Logging for these field names after a test API call
- [ ] **AC-3:** `AuditLog` table protected by row security (EP-DATA US-003); application role cannot DELETE or UPDATE entries — INSERT only
- [ ] **AC-4:** Audit log query API: `GET /api/v1/admin/audit?user_id={}&entity_type={}&from={}&to={}` returns paginated results; PHI fields in entity metadata masked (shown as `[REDACTED]`) for non-compliance roles
- [ ] **AC-5:** Integration test: `GET /api/v1/patients/{id}` with nurse JWT → verify 1 AuditLog entry with `action=READ, entity_type=PATIENT, entity_id={id}, user_id={nurse_id}` created within 100ms

## Definition of Done

- [ ] PHI fields confirmed absent from Cloud Logging after 20 test API calls with various PHI-bearing responses
- [ ] AuditLog DELETE attempt fails for api_writer role (confirmed via psql)
- [ ] Integration test passes for audit entry creation on every PHI endpoint

## Dependencies

- EP-DATA US-001 (AuditLog ORM model and row security policy)
- EP-011 US-001 (JWT for user_id extraction)
