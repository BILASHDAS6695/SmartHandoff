# US-001: Staff SSO Login with OIDC and MFA Enforcement

> **Epic:** EP-011 — Security, Auth & HIPAA Compliance | **Sprint:** 1 | **Points:** 5 | **Priority:** Must Have

## User Story

**As a** staff member,
**I want** to log in using the hospital's existing SSO with my work credentials and MFA,
**so that** I don't need a separate SmartHandoff password and access is as secure as the hospital's identity standards.

## Acceptance Criteria

- [ ] **AC-1:** Angular `AuthGuard` redirects unauthenticated users to hospital SSO OIDC authorization endpoint; successful login returns `id_token` + `access_token` via authorization code flow
- [ ] **AC-2:** FastAPI `/api/v1/auth/token` endpoint: receives `id_token`, fetches OIDC JWKS (1h in-memory cache), validates signature + expiry, verifies `amr` claim contains `mfa` → issues app JWT (15-min expiry, in-memory on client only)
- [ ] **AC-3:** Missing or invalid `amr: mfa` claim → 401 Unauthorized with `WWW-Authenticate: Bearer error="mfa_required"`
- [ ] **AC-4:** App JWT includes: `sub` (user_id), `email`, `role`, `unit` (nurse's assigned unit), `iat`, `exp`; JWT signed with RS256 key from Secret Manager
- [ ] **AC-5:** Silent token renewal: Angular refreshes app JWT every 12 minutes using OIDC `refresh_token` without user interruption
- [ ] **AC-6:** `AppUser` record created or updated on first login: `last_login`, `email`, `idp_subject`, `role` synced from JWT claims; `AuditLog` entry created for LOGIN action

## Definition of Done

- [ ] End-to-end login tested against hospital SSO sandbox with real MFA challenge
- [ ] JWT in-memory storage confirmed: DevTools shows no token in localStorage or sessionStorage
- [ ] 401 returned when MFA not in `amr` claim (test with modified token)

## Dependencies

- EP-TECH US-004 (JWT signing key in Secret Manager)
- EP-DATA US-001 (AppUser and AuditLog tables)
