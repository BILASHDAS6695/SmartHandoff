# US-004: Generate Context-Aware Handoff Checklists per Transition

> **Epic:** EP-003 — AI Agent Orchestration Framework | **Sprint:** 2 | **Points:** 5 | **Priority:** Should Have

## User Story

**As a** floor nurse,
**I want** a patient-specific handoff checklist generated automatically for each transfer or discharge,
**so that** I don't have to remember all required handoff steps — they're tailored to this patient's diagnosis and unit.

## Acceptance Criteria

- [ ] **AC-1:** On A02 (Transfer) or A03 (Discharge): Coordinator Agent calls Vertex AI Gemini with a prompt that includes patient diagnosis (ICD-10 codes), source unit, destination unit, and transition type — generating a structured checklist
- [ ] **AC-2:** Checklist contains at minimum: patient-specific items derived from diagnosis (e.g., "Diabetic patient: confirm insulin orders reconciled"), unit-specific items (e.g., "ICU → Ward: confirm monitoring order downgrade"), and standard items (e.g., "Confirm allergy band applied")
- [ ] **AC-3:** Checklist persisted as `Document` record with `document_type=HANDOFF_CHECKLIST`; surfaced in nurse's task queue via SignalR
- [ ] **AC-4:** Checklist rendered in Angular dashboard under Patient Detail → Tasks tab with checkable boxes; completion state persisted to DB
- [ ] **AC-5:** Minimum 3 patient-specific items generated for a test scenario (diabetic patient transferred from ICU to General Ward)

## Definition of Done

- [ ] Gemini-generated checklists validated by a clinical nurse for appropriate item quality in UAT
- [ ] Checklist completion persisted and visible in AgentTask detail

## Dependencies

- EP-003 US-001 (AgentTask framework)
- EP-003 US-003 (SignalR push for checklist delivery)
- EP-002 US-001 (FHIR client for diagnosis data)
