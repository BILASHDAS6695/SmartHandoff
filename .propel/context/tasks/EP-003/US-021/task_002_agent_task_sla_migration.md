# TASK-002: Add `sla_threshold_minutes` and `sla_breached` Columns to `AgentTask` via Alembic Migration

> **Story:** US-021 | **Epic:** EP-003 | **Sprint:** 2 | **Layer:** Backend | **Est:** 2 h
> **Status:** Draft | **Date:** 2026-07-16

---

## Context

US-021 Scenario 2 and DoD require the task status API to return `sla_threshold_minutes` and `sla_breached` fields per `AgentTask`. The `AgentTask` ORM model created in US-006/TASK-005 does not yet include these columns.

This task:
1. Updates the `AgentTask` SQLAlchemy model with the two new columns.
2. Generates and applies an Alembic migration that adds the columns to the `agent_task` table.
3. Adds a partial index on `(status, created_at)` to support the SLA monitor's polling query (TASK-003) without a full-table scan.

The `sla_breached` flag is written by the `SLAMonitor` (TASK-003) using the write session â€” this task only establishes the schema.

| US-021 AC | Requirement |
|---|---|
| **Scenario 2** | API response includes `sla_threshold_minutes` and `sla_breached` fields per task |
| **DoD** | `sla_breached` boolean flag updated on `AgentTask` record when threshold exceeded |

## Implementation Steps

### 1. Update `backend/app/models/agent_task.py`

Add the two new columns to the existing `AgentTask` model. Do **not** rewrite the entire file â€” apply a surgical change to the column definitions block only.

Locate the `AgentTask` class in `backend/app/models/agent_task.py` and add the following two columns after the `completed_time` column:

```python
    # SLA fields â€” populated by SLAMonitor (app/monitor/sla_monitor.py, US-021)
    sla_threshold_minutes: Mapped[int | None] = mapped_column(
        sa.Integer,
        nullable=True,
        comment="SLA threshold in minutes for this agent type (from sla_config.yaml).",
    )
    sla_breached: Mapped[bool] = mapped_column(
        sa.Boolean,
        nullable=False,
        server_default=sa.text("false"),
        comment="Set to TRUE by SLAMonitor when task exceeds its SLA threshold.",
    )
```

### 2. Add SLA Monitor Query Index

Add a partial index inside the `__table_args__` tuple of `AgentTask` to support the monitor's `WHERE status IN ('IN_PROGRESS', 'PENDING')` poll query (TASK-003):

```python
    __table_args__ = (
        # ... existing indexes ...
        sa.Index(
            "ix_agent_task_active_status_created",
            "status",
            "created_at",
            postgresql_where=sa.text("status IN ('IN_PROGRESS', 'PENDING')"),
        ),
    )
```

### 3. Generate Alembic Migration

From the `backend/` directory:

```bash
alembic revision --autogenerate \
  -m "add_sla_columns_to_agent_task"
```

### 4. Author the Migration Body

Verify the auto-generated file in `backend/alembic/versions/` and ensure the `upgrade()` and `downgrade()` functions match the following:

```python
"""add_sla_columns_to_agent_task

Revision ID: <auto-generated>
Revises: <previous revision>
Create Date: 2026-07-16

Adds sla_threshold_minutes and sla_breached columns to agent_task table.
Adds partial index to support SLAMonitor active-task polling query.

US-021: SLA monitoring requires these fields for breach detection and API response.
"""
from alembic import op
import sqlalchemy as sa


def upgrade() -> None:
    op.add_column(
        "agent_task",
        sa.Column(
            "sla_threshold_minutes",
            sa.Integer(),
            nullable=True,
            comment="SLA threshold in minutes for this agent type.",
        ),
    )
    op.add_column(
        "agent_task",
        sa.Column(
            "sla_breached",
            sa.Boolean(),
            nullable=False,
            server_default=sa.text("false"),
            comment="Set to TRUE by SLAMonitor when task exceeds its SLA threshold.",
        ),
    )
    op.create_index(
        "ix_agent_task_active_status_created",
        "agent_task",
        ["status", "created_at"],
        postgresql_where=sa.text("status IN ('IN_PROGRESS', 'PENDING')"),
    )


def downgrade() -> None:
    op.drop_index(
        "ix_agent_task_active_status_created",
        table_name="agent_task",
    )
    op.drop_column("agent_task", "sla_breached")
    op.drop_column("agent_task", "sla_threshold_minutes")
```

### 5. Apply the Migration (Dev Environment)

```bash
alembic upgrade head
```

Verify the columns exist:
