# US-001: Coordinator Agent Creates All Agent Tasks Within 2 Seconds of ADT Event

> **Epic:** EP-003 — AI Agent Orchestration Framework | **Sprint:** 2 | **Points:** 8 | **Priority:** Must Have

## User Story

**As a** care coordinator,
**I want** the Transition Coordinator Agent to subscribe to ADT events and create all downstream agent tasks within 2 seconds,
**so that** care transition workflows begin immediately after an admission, transfer, or discharge — with no manual triggering required.

## Acceptance Criteria

- [ ] **AC-1:** Coordinator Agent subscribes to `adt-events` Pub/Sub topic via `coordinator-sub` subscription; message delivery triggers workflow creation within 2 seconds of Pub/Sub publish
- [ ] **AC-2:** On A01 (Admit): creates `AgentTask` records for all 5 agents: Documentation, MedRecon, BedMgmt, Followup, Comms — each with `status=PENDING`, `agent_type`, `encounter_id`, `start_time`
- [ ] **AC-3:** On A02 (Transfer): creates AgentTask records for Documentation and BedMgmt agents only
- [ ] **AC-4:** On A03 (Discharge): creates AgentTask records for all 5 agents with discharge-specific metadata
- [ ] **AC-5:** `GET /api/v1/encounters/{id}/tasks` returns all AgentTask records with current status within 200ms
- [ ] **AC-6:** Integration test: send A01 via HL7 → measure time from Pub/Sub publish to all 5 AgentTask rows in DB → must be ≤2 seconds in staging environment

## Definition of Done

- [ ] Integration test confirms ≤2 second task creation measured end-to-end
- [ ] All 5 agent task types created with correct metadata for A01/A02/A03 events
- [ ] Cloud Run service deployed with min-instances=1 to avoid cold start latency

## Dependencies

- EP-001 US-004 (Pub/Sub events published)
- EP-DATA US-001 (AgentTask ORM model)
- EP-TECH US-001 (Pub/Sub subscriptions + DLQ configured)
