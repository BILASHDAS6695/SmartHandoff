# US-002: SLA Monitoring and Supervisor Escalation for Overdue Tasks

> **Epic:** EP-003 — AI Agent Orchestration Framework | **Sprint:** 2 | **Points:** 3 | **Priority:** Must Have

## User Story

**As a** supervisor,
**I want** overdue agent tasks automatically escalated to me after 30 minutes with no completion,
**so that** I can intervene before delays impact patient care or regulatory compliance.

## Acceptance Criteria

- [ ] **AC-1:** Coordinator Agent runs a background SLA checker every 5 minutes; identifies `AgentTask` records where `status IN (PENDING, IN_PROGRESS)` and `start_time < NOW() - INTERVAL '30 minutes'`
- [ ] **AC-2:** For each overdue task: publishes escalation message to `notification-requests` Pub/Sub topic with `notification_type=SLA_BREACH`, `agent_type`, `encounter_id`, `elapsed_minutes`
- [ ] **AC-3:** SLA thresholds configurable per agent type via Secret Manager environment variable (default: 30 minutes for all); Medication Reconciliation SLA = 24 hours (BR-002)
- [ ] **AC-4:** Escalation notification delivered to supervisor dashboard via SignalR AND to supervisor SMS/email via Notification Service
- [ ] **AC-5:** `AgentTask.retry_count` incremented and task status set to `ESCALATED`; task remains in queue for manual resolution
- [ ] **AC-6:** Integration test: create AgentTask with start_time 31 minutes ago → verify escalation published to Pub/Sub

## Definition of Done

- [ ] SLA checker tested with synthetic overdue tasks in staging
- [ ] Escalation notification confirmed in supervisor dashboard and SMS

## Dependencies

- EP-003 US-001 (AgentTask records exist)
- EP-013 US-001 (Notification Service for SMS/email delivery)
