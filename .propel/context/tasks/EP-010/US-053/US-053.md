# US-053: Display Structured Discharge Instructions with Language Switcher

> **Epic:** EP-010 — Patient Portal | **Sprint:** 2 | **Points:** 5 | **Priority:** Must Have
> **Status:** Draft | **Date:** 2026-07-14

---

## User Story

**As a** patient (Pat) who speaks French,
**I want to** see my discharge instructions displayed in French in clearly structured sections — medications, activity, diet, follow-up, warning signs — with a language switcher to change languages,
**So that** I can understand my care plan without relying on an interpreter or struggling with English medical text.

---

## Business Value

Joint Commission standard RC.02.04.01 requires instructions in the patient's language. Non-English-speaking patients who cannot understand their discharge instructions are 35% more likely to return to the ED within 30 days. Multilingual structured instructions directly reduce preventable readmissions and satisfy regulatory requirements.

---

## Acceptance Criteria

### Scenario 1: Instructions displayed in patient's preferred language on load
**Given** a patient with `preferred_language=fr` accesses the portal
**When** the discharge instructions page loads
**Then** all instruction text is rendered in French; the language switcher shows `FR` as the active language.

### Scenario 2: Language switcher changes all text instantly
**Given** a patient is viewing instructions in English
**When** they select "Español" from the language switcher
**Then** all instruction section text changes to Spanish within 500ms (client-side language switch — no new API call needed if translations already loaded).

### Scenario 3: Sections clearly structured with visual hierarchy
**Given** the discharge instructions page renders
**When** the patient views the page on a mobile device (375px viewport)
**Then** sections are clearly demarcated: "Your Medications" (pill icon), "Activity" (walking icon), "Diet" (fork icon), "Follow-up Appointments" (calendar icon), "Warning Signs" (warning icon with prominent red border).

### Scenario 4: Warning signs section is visually prominent
**Given** the instructions include warning signs (e.g. "Call 911 if chest pain returns")
**When** the patient scrolls to the Warning Signs section
**Then** the section has a red border, amber background, and a "⚠ Call 911 immediately if you experience:" header — visually distinct from other sections.

## Definition of Done

- [ ] `DischargeInstructionsComponent`: sections mapped from `Document.content` structured JSON
- [ ] Language switcher: `MatButtonToggle` group with supported languages from `Document.translations`; updates `activeLanguage` signal
- [ ] Section icons: Angular Material `mat-icon` with healthcare-specific icons
- [ ] Warning signs section: custom Angular directive `appWarningSection` applies red border + amber background
- [ ] Mobile-first responsive: single-column stacked layout below 768px
- [ ] Touch target ≥44px for language switcher and all interactive elements
- [ ] `axe-core` WCAG 2.1 AA test on portal instructions page
- [ ] Code reviewed and approved

---

## Technical Notes

- `Document.translations` JSONB: `{"en": {...sections}, "es": {...sections}, "fr": {...sections}, ...}`
- Language switch: computed signal `currentContent = computed(() => translations[activeLanguage()] ?? translations['en'])`
- Angular Signals preferred over traditional `BehaviorSubject` for language switching
- Test on real devices: iOS Safari 16+ and Android Chrome 120+ (primary patient devices)
