# US-004: Offline Access to Discharge Instructions via Service Worker

> **Epic:** EP-010 — Patient Portal | **Sprint:** 2 | **Points:** 3 | **Priority:** Should Have

## User Story

**As a** patient with unreliable internet at home,
**I want** my discharge instructions to load from my phone even when I'm offline,
**so that** I can check my medication schedule or warning signs without needing a mobile data connection.

## Acceptance Criteria

- [ ] **AC-1:** Angular Service Worker (`@angular/pwa`) configured to cache: portal app shell (HTML/CSS/JS), patient discharge instructions (API response from `GET /portal/instructions`), and medication summary document
- [ ] **AC-2:** On first authenticated portal visit: service worker pre-fetches and caches instructions under `Cache-API`; cache TTL = 30 days post-discharge (after which cache is invalidated)
- [ ] **AC-3:** Offline mode: portal loads from cache when `navigator.onLine=false`; offline indicator banner shown ("You're offline — showing saved instructions from {cached_date}"); chatbot disabled with "Chat requires internet connection" message
- [ ] **AC-4:** PWA installability: `manifest.json` configured with hospital name, icon, `display=standalone`; "Add to Home Screen" prompt visible on iOS Safari and Android Chrome
- [ ] **AC-5:** Cache invalidated on service worker update (new portal deployment) — patient sees "Instructions updated" banner on reconnect

## Definition of Done

- [ ] Offline test: load portal, kill network in Chrome DevTools → instructions still visible from cache
- [ ] PWA install tested on iOS Safari and Android Chrome
- [ ] Service worker cache TTL verified: 30 days post-discharge date

## Dependencies

- EP-010 US-002 (Instructions API endpoint defined)
- EP-009 US-001 (Angular PWA Service Worker configured in scaffold)
