# US-054: Enable PDF Download and Offline Instruction Caching

> **Epic:** EP-010 — Patient Portal | **Sprint:** 2 | **Points:** 3 | **Priority:** Should Have
> **Status:** Draft | **Date:** 2026-07-14

---

## User Story

**As a** patient,
**I want to** download my discharge instructions as a PDF and have the portal work offline so I can refer to my care plan even in areas without internet access,
**So that** I can access my instructions when I'm home without Wi-Fi, or share them with a family caregiver.

---

## Business Value

Elderly patients and caregivers often prefer printed or downloaded instructions. PDF download removes the dependency on hospital portals for ongoing reference. Service Worker offline caching ensures patients in rural areas with poor connectivity can still access their instructions — directly improving medication adherence and follow-up compliance.

---

## Acceptance Criteria

### Scenario 1: PDF download produces readable, formatted file
**Given** a patient clicks "Download PDF" on the instructions page
**When** the PDF is generated client-side
**Then** a formatted PDF downloads with: patient first name, discharge date, hospital name, all instruction sections with clear headings, and a footer reading "Generated by SmartHandoff — Always contact your care team with questions."

### Scenario 2: Portal instructions load from cache when offline
**Given** a patient previously viewed their discharge instructions with an internet connection
**When** they open the portal while offline (Chrome DevTools offline mode)
**Then** the instructions page loads from Service Worker cache within 2 seconds; an offline banner "You're viewing cached instructions" is displayed.

### Scenario 3: Service Worker caches instructions for 30 days post-discharge
**Given** a patient's discharge was 15 days ago
**When** the Service Worker cache is inspected
**Then** the discharge instructions JSON response is still in the cache (cache TTL = 30 days from discharge date); the portal renders from cache without network.

### Scenario 4: PWA is installable on Android and iOS
**Given** a patient visits the portal on Android Chrome
**When** the PWA install prompt appears
**Then** the app installs successfully to the home screen; the installed app launches full-screen without browser chrome; the `manifest.json` has correct `name`, `short_name`, `icons`, and `display=standalone`.

---

## Definition of Done

- [ ] Client-side PDF generation using `jsPDF` library: formatted layout with SmartHandoff header
- [ ] Angular Service Worker: `ngsw-config.json` caches discharge instructions API response for 30 days
- [ ] Cache TTL: calculated as `discharge_date + 30 days`; expired cache cleared by SW background sync
- [ ] Offline banner: Angular Service Worker online/offline event listener → `MatBanner`
- [ ] `manifest.json`: `display=standalone`, app icons (192px and 512px), `start_url=/portal`
- [ ] PWA install prompt: `BeforeInstallPromptEvent` handled, "Add to Home Screen" button shown
- [ ] Unit tests: PDF content validation, SW cache registration
- [ ] Code reviewed and approved

---

## Technical Notes

- `jsPDF` 2.x with `html2canvas` for section rendering, or `jsPDF.autoTable` for structured table layout
- Service Worker: Angular `@angular/service-worker` (built-in) with `precacheStrategy: 'freshness'` for API responses
- PHI in PDF: include only `first_name` (not full name, DOB, MRN) per HIPAA minimum-necessary
- PDF filename: `SmartHandoff_Discharge_Instructions_{first_name}_{discharge_date}.pdf`
