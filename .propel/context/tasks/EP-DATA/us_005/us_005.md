# US-005: Implement Data Retention Automation via pg_cron

> **Epic:** EP-DATA — Data Foundation & Schema | **Sprint:** 1 | **Points:** 2 | **Priority:** Must Have

## User Story

**As a** compliance officer,
**I want** data retention policies enforced automatically by scheduled database jobs,
**so that** patient records are archived after 7 years and audit logs are retained for 6 years without manual intervention.

## Acceptance Criteria

- [ ] **AC-1:** pg_cron job `archive_old_encounters` runs weekly: sets `encounter.deleted_at = NOW()` for encounters where `discharge_date < NOW() - INTERVAL '7 years'`; exports archived records to Cloud Storage before marking
- [ ] **AC-2:** pg_cron job `audit_log_retention_check` runs monthly: verifies no `audit_log` records older than 6 years have been deleted (compliance check); logs result to Cloud Logging
- [ ] **AC-3:** `cron.job` table in PostgreSQL shows both jobs scheduled with correct cron expressions; job history visible in `cron.job_run_details`
- [ ] **AC-4:** Archived encounter data exported in JSON Lines format to `gs://smarthandoff-archive/encounters/{year}/` before soft-delete
- [ ] **AC-5:** Soft-deleted records excluded from all ORM default query scopes; explicitly queryable only via `include_deleted=True` parameter (admin only)

## Definition of Done

- [ ] Jobs visible in `cron.job` table in dev environment
- [ ] Manual trigger of archive job verified in staging; Cloud Storage export confirmed
- [ ] Compliance check job produces correct output for mixed-age test data

## Dependencies

- EP-DATA US-001 (encounter and audit_log tables exist)
- EP-TECH US-001 (Cloud Storage archive bucket provisioned)
