# US-002: ML-Predicted Discharge Times for Proactive Bed Planning

> **Epic:** EP-006 — Bed Management Agent | **Sprint:** 2 | **Points:** 8 | **Priority:** Should Have

## User Story

**As a** bed manager,
**I want** ML-predicted discharge times shown for every occupied bed,
**so that** I can proactively assign incoming patients to beds that will be available soon — rather than waiting for a discharge to happen.

## Acceptance Criteria

- [ ] **AC-1:** Scikit-learn GradientBoostingRegressor model trained on historical encounter data: features = admit_date, diagnosis_category (ICD-10 chapter), patient_age, admission_type, unit; target = length_of_stay_hours
- [ ] **AC-2:** Model serialised to `joblib` format, stored in Cloud Storage `gs://smarthandoff-models/bed_los_v{version}.joblib`; loaded into ML Inference Service at startup (no per-request disk access)
- [ ] **AC-3:** ML Inference Service `POST /predict/los` accepts `{encounter_id, admit_date, diagnosis_codes, patient_age, unit}` → returns `{predicted_discharge_at, confidence_interval_hours}`; p95 latency <500ms
- [ ] **AC-4:** Predicted discharge time ±2 hours accuracy verified on holdout test set (20% of training data); MAE ≤2.0 hours
- [ ] **AC-5:** `Bed.predicted_discharge_at` updated every hour by Bed Management Agent calling ML Inference Service for all currently-occupied beds; Bed Board shows prediction in bed card footer
- [ ] **AC-6:** Bed assignment recommendation `GET /api/v1/beds/recommend?encounter_id={id}` returns top 3 beds ranked by: best fit for patient needs + earliest predicted vacancy

## Definition of Done

- [ ] Model MAE ≤2.0 hours verified on holdout dataset
- [ ] ML Inference Service p95 <500ms confirmed under 50 concurrent prediction requests
- [ ] Bed board shows predicted discharge times for 10 test occupied beds in staging

## Dependencies

- EP-006 US-001 (Bed board and Bed ORM established)
- EP-TECH US-001 (ML Inference Cloud Run service provisioned)
