# US-001: Real-Time Bed Availability Board with ADT Event Updates

> **Epic:** EP-006 — Bed Management Agent | **Sprint:** 2 | **Points:** 5 | **Priority:** Must Have

## User Story

**As a** bed manager,
**I want** a real-time visual bed board that automatically updates bed status on every ADT event,
**so that** I can manage patient flow without phone calls or manual status tracking.

## Acceptance Criteria

- [ ] **AC-1:** Bed Management Agent subscribes to `adt-events`; on A01 → marks assigned bed `OCCUPIED`; on A02 → marks source bed `DIRTY`, destination `OCCUPIED`; on A03 → marks discharged patient's bed `DIRTY`; all updates within 5 seconds of ADT event
- [ ] **AC-2:** `mv_bed_board` materialised view refreshed (CONCURRENTLY) after each bed status change; refresh must complete within 60 seconds
- [ ] **AC-3:** Bed Board API `GET /api/v1/beds?unit={}&status={}&type={}` returns filtered bed records from `mv_bed_board`; p95 response time <200ms under 100 concurrent requests
- [ ] **AC-4:** Angular Bed Board screen (`/beds`) renders a visual grid grouped by unit; bed cards colour-coded: green=CLEAN, amber=DIRTY, red=OCCUPIED, grey=BLOCKED/MAINTENANCE; discharge prediction time shown on occupied beds
- [ ] **AC-5:** SignalR event `BedStatusChanged` broadcast to `role-BED_MANAGER` group on every bed update; Angular updates bed card without page reload
- [ ] **AC-6:** Bed inventory initialised from a JSON seed file (unit, room, bed_label, bed_type) uploaded to Secret Manager; new hospital deployments configurable without code changes

## Definition of Done

- [ ] Bed board UI tested end-to-end in staging: send A01 → bed turns red within 60 seconds
- [ ] Performance test confirms p95 <200ms for bed board API under 100 concurrent users

## Dependencies

- EP-001 US-004 (ADT events in Pub/Sub)
- EP-DATA US-004 (mv_bed_board materialised view)
- EP-003 US-003 (SignalR push)
