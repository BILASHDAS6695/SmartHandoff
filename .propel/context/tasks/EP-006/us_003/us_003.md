# US-003: Fire ED Boarding Alert When Patient Waits More Than 2 Hours for a Bed

> **Epic:** EP-006 — Bed Management Agent | **Sprint:** 2 | **Points:** 3 | **Priority:** Must Have

## User Story

**As an** ED charge nurse,
**I want** an automated alert when a patient has waited more than 2 hours for an inpatient bed assignment,
**so that** I can escalate before the ED becomes overwhelmed and patient care is compromised.

## Acceptance Criteria

- [ ] **AC-1:** Bed Management Agent maintains an in-memory timer per pending admission (A01 received, no bed assigned yet); at 2-hour threshold: publishes `ED_BOARDING_ALERT` to `notification-requests` Pub/Sub topic
- [ ] **AC-2:** Alert delivered via SignalR push to `role-BED_MANAGER` and `unit-ED` groups within 60 seconds of 2-hour threshold; alert contains: patient encounter ID (masked MRN), time waiting, available beds count
- [ ] **AC-3:** Bed Board screen shows a red "BOARDING" badge on the patient card once alert is fired; badge cleared on bed assignment
- [ ] **AC-4:** Bed manager can dismiss alert and trigger boarding resolution workflow: bed board highlights beds with earliest predicted discharge times; assignment made via `PATCH /api/v1/beds/{id}/assign?encounter_id={id}`
- [ ] **AC-5:** Alert duration (from fire to resolution) recorded in `alert_log` table for KPI reporting (`mv_kpi_daily` includes avg_boarding_time metric)
- [ ] **AC-6:** Integration test: create A01 event with no bed assignment → advance time to 120 minutes → verify alert fired exactly at threshold

## Definition of Done

- [ ] Integration test confirms alert fires at exactly 2-hour mark (using time-injection test utility)
- [ ] SignalR alert received in browser dashboard within 60 seconds in end-to-end test

## Dependencies

- EP-006 US-001 (Bed board and bed assignment tracking)
- EP-013 US-001 (Notification Service for SMS alert delivery)
- EP-003 US-003 (SignalR push)
