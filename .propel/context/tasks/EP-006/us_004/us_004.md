# US-004: Automatically Notify Housekeeping on Patient Discharge

> **Epic:** EP-006 — Bed Management Agent | **Sprint:** 2 | **Points:** 2 | **Priority:** Should Have

## User Story

**As a** bed manager,
**I want** housekeeping automatically notified when a patient is discharged,
**so that** beds are cleaned and ready for the next patient without a manual call or message from nursing.

## Acceptance Criteria

- [ ] **AC-1:** On A03 (Discharge) event: Bed Management Agent publishes `HOUSEKEEPING_NOTIFICATION` message to `notification-requests` Pub/Sub within 5 seconds of event processing; message contains: unit, room, bed_label, discharge_time
- [ ] **AC-2:** Notification Service sends SMS/email to the configured housekeeping contact(s) for the unit (stored in Secret Manager as a per-unit JSON map: `{"ICU": "+1555...", "Ward-A": "+1555..."}`
- [ ] **AC-3:** `Bed.status` updated to `DIRTY` (not `CLEAN`) immediately on A03; remains `DIRTY` until manually marked clean via `PATCH /api/v1/beds/{id}/status {status: CLEAN}` by housekeeping or nurse
- [ ] **AC-4:** Time from discharge (A03 event) to bed marked CLEAN recorded in `Bed.last_updated` for turnaround time KPI tracking
- [ ] **AC-5:** If housekeeping notification fails to deliver (SMS error): retry 3× with exponential backoff; undeliverable → nurse for that unit alerted via SignalR

## Definition of Done

- [ ] End-to-end test: A03 event → housekeeping SMS received within 30 seconds in staging
- [ ] Bed board shows DIRTY status immediately after discharge

## Dependencies

- EP-006 US-001 (Bed status tracking)
- EP-013 US-001 (Notification Service SMS delivery)
