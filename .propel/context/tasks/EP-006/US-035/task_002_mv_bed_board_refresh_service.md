# TASK-002: mv_bed_board CONCURRENTLY Refresh Service and Unique Index Migration

> **Story:** US-035 | **Epic:** EP-006 | **Sprint:** 2 | **Layer:** Backend | **Est:** 2 h
> **Status:** Draft | **Date:** 2026-07-17

---

## Context

US-035 requires the `mv_bed_board` materialised view to be refreshed **on-demand** after every bed status change (not only on the pg_cron 60-second schedule established by US-009). PostgreSQL `REFRESH MATERIALIZED VIEW CONCURRENTLY` avoids locking the view during the refresh, but it requires a **unique index** on the materialised view.

This task:
1. Adds an Alembic migration to create a unique index on `mv_bed_board` if one does not already exist from US-009.
2. Implements `BedBoardRefreshService` — a lightweight async service that issues the `CONCURRENTLY` refresh and is called by `BedManagementAgent` after each bed status write (TASK-001).

**Design references:**
- US-035 Technical Notes — "trigger `REFRESH MATERIALIZED VIEW CONCURRENTLY mv_bed_board` in a background async task"; CONCURRENTLY requires unique index
- design.md §6.3 (DR-007) — `mv_bed_board` refresh every 60 s (scheduled); this adds event-driven refresh
- design.md §5.2 (TR-010) — read replica routing; materialised views refreshed every 60 s as baseline

## Implementation Steps

### 1. Add Alembic migration — unique index on mv_bed_board

Check whether US-009 migration already creates a unique index. If not, create a new migration:

```bash
cd backend
alembic revision --message "add_unique_idx_mv_bed_board_bed_id"
```

Migration file content:

```python
"""add_unique_idx_mv_bed_board_bed_id

Revision ID: <auto-generated>
Revises: <us-009-revision-id>
Create Date: 2026-07-17

CONCURRENTLY refresh of mv_bed_board requires a unique index.
Design ref: US-035 Technical Notes
"""
from alembic import op


def upgrade() -> None:
    # Unique index required for REFRESH MATERIALIZED VIEW CONCURRENTLY.
    # CREATE UNIQUE INDEX CONCURRENTLY cannot run inside a transaction block,
    # so we set transaction=False on this migration.
    op.execute(
        "CREATE UNIQUE INDEX IF NOT EXISTS uix_mv_bed_board_bed_id "
        "ON mv_bed_board (bed_id)"
    )


def downgrade() -> None:
    op.execute("DROP INDEX IF EXISTS uix_mv_bed_board_bed_id")
```

> **Important:** Set `transaction=False` in the migration's `alembic.ini` override OR use
> `op.execute` without wrapping in `BEGIN`/`COMMIT` manually, since
> `CREATE INDEX CONCURRENTLY` cannot run inside an explicit transaction.
> Use Alembic's `non_transactional_ddl` pattern:
>
> ```python
> def upgrade() -> None:
>     with op.get_context().autocommit_block():
>         op.execute(
>             "CREATE UNIQUE INDEX CONCURRENTLY IF NOT EXISTS "
>             "uix_mv_bed_board_bed_id ON mv_bed_board (bed_id)"
>         )
> ```

### 2. Implement `backend/app/agents/bed_management/refresh_service.py`

```python
"""BedBoardRefreshService — issues CONCURRENTLY refresh of mv_bed_board.

Called by BedManagementAgent (TASK-001) immediately after each bed status
write to ensure the materialised view is updated within the 60-second SLA
(US-035 AC Scenarios 1 and 2).

Design refs:
    US-035 Technical Notes — REFRESH MATERIALIZED VIEW CONCURRENTLY mv_bed_board
    US-035 AC Scenario 1   — board shows OCCUPIED within 60 s of A01
    US-035 AC Scenario 2   — board shows DIRTY within 60 s of A03
    design.md §6.3 DR-007  — mv_bed_board baseline refresh every 60 s
"""
from __future__ import annotations

import asyncio
import logging
from typing import Any

from sqlalchemy.ext.asyncio import AsyncSession

logger = logging.getLogger(__name__)

_REFRESH_SQL = "REFRESH MATERIALIZED VIEW CONCURRENTLY mv_bed_board"


class BedBoardRefreshService:
    """Issues a CONCURRENTLY materialised-view refresh in a background task.

    The refresh is fired-and-forgotten via ``asyncio.create_task`` so it does
    not block the agent's Pub/Sub acknowledgement path.

    Args:
        write_session_factory: Async SQLAlchemy session factory bound to the
            primary DB (required — REFRESH MV must run on primary, not replica).
    """

    def __init__(self, write_session_factory: Any) -> None:
        self._factory = write_session_factory

    async def refresh_async(self) -> None:
        """Schedule a background CONCURRENTLY refresh without blocking the caller."""
        asyncio.create_task(self._do_refresh(), name="mv_bed_board_refresh")

    async def refresh_sync(self) -> None:
        """Execute the refresh and await completion (used during startup seeding)."""
        await self._do_refresh()

    async def _do_refresh(self) -> None:
        """Execute REFRESH MATERIALIZED VIEW CONCURRENTLY mv_bed_board."""
        try:
            async with self._factory() as session:
                await session.execute(_REFRESH_SQL)
                await session.commit()
            logger.info("mv_bed_board CONCURRENTLY refresh completed")
        except Exception:
            # Refresh failure is non-fatal — pg_cron will retry within 60 s
            logger.exception("mv_bed_board CONCURRENTLY refresh failed (non-fatal)")
```
