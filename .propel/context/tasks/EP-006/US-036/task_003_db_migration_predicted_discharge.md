# TASK-003: DB Migration — predicted_discharge_time Column and mv_bed_board View Update

> **Story:** US-036 | **Epic:** EP-006 | **Sprint:** 2 | **Layer:** Backend / Database | **Est:** 2 h
> **Status:** Draft | **Date:** 2026-07-17

---

## Context

US-036 requires that the discharge time prediction be persisted on the `encounter` record and projected into the `mv_bed_board` materialised view so the bed board UI can display it.

This task:
1. Adds an Alembic migration that extends the `encounter` table with a nullable `predicted_discharge_time` column and a `discharge_prediction_confidence` column (to persist the confidence level from the ML Inference Service response).
2. Updates the `mv_bed_board` materialised view definition to include these two columns.

The `encounter` ORM model is updated accordingly.

**Design references:**
- US-036 DoD — "Prediction stored in `encounter.predicted_discharge_time` and reflected in `mv_bed_board`"
- US-036 AC Scenario 4 — bed board shows `predicted_discharge_time` + colour-coded confidence indicator
- design.md §6.1 (DR-007) — `mv_bed_board`: real-time bed status (refresh every 60 s)
- design.md §6.1 (DR-002) — No PHI in the new columns (both are predictions, not identifiers)
- design.md §5.2 (TR-010) — read replica routing; materialised view used for all dashboard reads

### 1. Create Alembic migration

### 2. Implement the migration file

```python
"""add_predicted_discharge_time_to_encounter

Revision ID: <auto-generated>
Revises: <us-035-revision-id>
Create Date: 2026-07-17

Adds discharge time prediction columns to encounter and updates mv_bed_board.

Design refs:
    US-036 DoD — predicted_discharge_time + mv_bed_board projection
    US-036 AC Scenario 4 — confidence level on bed board
"""
from alembic import op
import sqlalchemy as sa


def upgrade() -> None:
    # 1. Extend encounter table
    op.add_column(
        "encounter",
        sa.Column(
            "predicted_discharge_time",
            sa.DateTime(timezone=True),
            nullable=True,
            comment="ML-predicted discharge datetime (UTC). NULL if not yet predicted.",
        ),
    )
    op.add_column(
        "encounter",
        sa.Column(
            "discharge_prediction_confidence",
            sa.String(10),
            nullable=True,
            comment="Confidence tier: 'high', 'medium', 'low', or NULL if unpredicted.",
        ),
    )
    op.add_column(
        "encounter",
        sa.Column(
            "discharge_prediction_interval_hours",
            sa.Numeric(precision=5, scale=2),
            nullable=True,
            comment="±hours confidence interval returned by ML Inference Service.",
        ),
    )

    # 2. Drop and recreate mv_bed_board to include prediction columns.
    #    CONCURRENTLY cannot be used on DROP; use a transaction-safe plain drop.
    op.execute("DROP MATERIALIZED VIEW IF EXISTS mv_bed_board")
    op.execute("""
        CREATE MATERIALIZED VIEW mv_bed_board AS
        SELECT
            b.id                            AS bed_id,
            b.unit,
            b.room,
            b.bed_number,
            b.status                        AS bed_status,
            e.id                            AS encounter_id,
            e.predicted_discharge_time,
            e.discharge_prediction_confidence,
            e.discharge_prediction_interval_hours,
            b.updated_at                    AS last_updated
        FROM bed b
        LEFT JOIN encounter e
            ON e.bed_id = b.id
            AND e.status = 'ADMITTED'
            AND e.deleted_at IS NULL
        WITH DATA
    """)

    # 3. Recreate unique index required for CONCURRENTLY refresh (US-035/TASK-002)
    op.execute(
        "CREATE UNIQUE INDEX IF NOT EXISTS uix_mv_bed_board_bed_id ON mv_bed_board (bed_id)"
    )


def downgrade() -> None:
    # Recreate view without prediction columns, then remove encounter columns
    op.execute("DROP MATERIALIZED VIEW IF EXISTS mv_bed_board")
    op.execute("""
        CREATE MATERIALIZED VIEW mv_bed_board AS
        SELECT
            b.id        AS bed_id,
            b.unit,
            b.room,
            b.bed_number,
            b.status    AS bed_status,
            e.id        AS encounter_id,
            b.updated_at AS last_updated
        FROM bed b
        LEFT JOIN encounter e
            ON e.bed_id = b.id
            AND e.status = 'ADMITTED'
            AND e.deleted_at IS NULL
        WITH DATA
    """)
    op.execute(
        "CREATE UNIQUE INDEX IF NOT EXISTS uix_mv_bed_board_bed_id ON mv_bed_board (bed_id)"
    )

    op.drop_column("encounter", "discharge_prediction_interval_hours")
    op.drop_column("encounter", "discharge_prediction_confidence")
    op.drop_column("encounter", "predicted_discharge_time")
```

### 3. Update `backend/app/models/encounter.py` — add new columns

```python
# Add to Encounter SQLAlchemy model class (alongside existing columns):

from sqlalchemy import Column, DateTime, Numeric, String
from sqlalchemy.orm import Mapped, mapped_column
from typing import Optional
from datetime import datetime
import decimal

# Inside class Encounter(Base):
predicted_discharge_time: Mapped[Optional[datetime]] = mapped_column(
    DateTime(timezone=True),
    nullable=True,
    comment="ML-predicted discharge datetime (UTC).",

    String(10),
    nullable=True,
    comment="Confidence tier: 'high', 'medium', 'low', or NULL.",

    Numeric(precision=5, scale=2),
    nullable=True,
```

### 4. Add index on encounter.predicted_discharge_time for dashboard queries

Extend the migration `upgrade()` function:

```python
# Partial index: only encountered with non-null predictions and ADMITTED status

    ON encounter (predicted_discharge_time)
```
