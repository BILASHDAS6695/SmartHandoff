# US-004: Dual-Pane Document Review UI with Inline Editing and Clinician Approval

> **Epic:** EP-004 — Documentation Agent | **Sprint:** 2 | **Points:** 5 | **Priority:** Must Have

## User Story

**As a** clinician,
**I want** to review AI-generated discharge documents in a dual-pane editor where I can read the AI draft and edit the final version side by side,
**so that** I can efficiently correct inaccuracies and approve the document with a full audit trail of my changes.

## Acceptance Criteria

- [ ] **AC-1:** Document Review screen (`/patients/:id/documents`) renders a dual-pane layout: left pane = AI-generated draft (read-only), right pane = editable version (rich text editor, e.g., ProseMirror or Quill)
- [ ] **AC-2:** Every keystroke in the right pane recorded as a change event: `{field, original_value, new_value, author_user_id, timestamp}` stored as JSON array in `Document.change_tracking`
- [ ] **AC-3:** "AI-Assisted — Review Required" badge displayed prominently on document until `Document.status=APPROVED`; badge disappears on approval
- [ ] **AC-4:** "Approve & Sign" button available only to users with `PHYSICIAN` role; clicking triggers: `PATCH /api/v1/documents/{id}/approve` → sets `Document.status=APPROVED`, `reviewed_by_user_id`, `approved_at`; creates `audit_log` entry with `action=APPROVE`
- [ ] **AC-5:** "Reject" button sends document back with a mandatory rejection reason; `Document.status=REJECTED`; Documentation Agent notified via Pub/Sub to regenerate or flag for manual completion
- [ ] **AC-6:** Non-physician role attempting to approve receives 403 Forbidden; approve button is hidden in Angular UI for non-physician roles

## Definition of Done

- [ ] UAT with 3 physicians: ≥80% rate review workflow as faster than manual documentation
- [ ] Change tracking JSON verified in DB after edit session
- [ ] RBAC tested: nurse account cannot call approve endpoint

## Dependencies

- EP-004 US-001 (Documents generated with PENDING_REVIEW status)
- EP-011 (RBAC — physician role required for approval)
- EP-003 US-003 (SignalR push for approval queue badge)
