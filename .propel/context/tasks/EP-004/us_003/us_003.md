# US-003: Translate Discharge Instructions into Patient's Preferred Language

> **Epic:** EP-004 — Documentation Agent | **Sprint:** 2 | **Points:** 3 | **Priority:** Should Have

## User Story

**As a** non-English-speaking patient,
**I want** my discharge instructions delivered in my preferred language,
**so that** I can understand my care plan without depending on a family member or interpreter.

## Acceptance Criteria

- [ ] **AC-1:** Agent reads `Patient.communication.language` from FHIR (ISO 639-1 code); if not English, generates instructions in patient language using Gemini translation prompt targeting: Spanish (`es`), French (`fr`), Mandarin (`zh`), Portuguese (`pt`)
- [ ] **AC-2:** Bilingual pair stored: `Document` records for both English and translated language, linked by `parent_document_id` field
- [ ] **AC-3:** Quality check: automated back-translation of translated text compared to English original; semantic delta <15% (measured via cosine similarity of Gemini embeddings); documents with delta ≥15% flagged for manual review
- [ ] **AC-4:** Patient portal auto-selects the patient's preferred language document; language switcher allows switching between available languages
- [ ] **AC-5:** If patient language not in supported set: instructions generated in English with a note "Translation to {language} not yet supported"

## Definition of Done

- [ ] All 4 non-English languages tested with real Gemini translation calls in staging
- [ ] Back-translation quality check verified for Spanish and Mandarin test cases

## Dependencies

- EP-004 US-002 (Patient instructions generated in English first)
- EP-002 US-001 (FHIR client for language preference)
