# US-002: Generate Patient-Friendly Discharge Instructions at ≤6th-Grade Reading Level

> **Epic:** EP-004 — Documentation Agent | **Sprint:** 2 | **Points:** 5 | **Priority:** Must Have

## User Story

**As a** discharged patient,
**I want** discharge instructions written in simple, everyday language I can understand,
**so that** I can follow my care plan correctly at home and avoid preventable readmissions.

## Acceptance Criteria

- [ ] **AC-1:** Separate LLM prompt generates patient instructions (distinct from the physician discharge summary) with explicit instruction to use simple words, short sentences, and avoid medical jargon
- [ ] **AC-2:** Flesch-Kincaid Grade Level calculation applied to generated English text; if grade level >6.0, agent makes one automatic re-generation request with a stricter simplification instruction
- [ ] **AC-3:** Instructions structured into clearly labelled sections: "Your Medications", "Activity", "Diet", "When to Call Your Doctor", "Your Follow-up Appointment"
- [ ] **AC-4:** Instructions persisted as separate `Document` record with `document_type=PATIENT_INSTRUCTIONS`; linked to same encounter as discharge summary
- [ ] **AC-5:** Patient instructions automatically attached to patient portal after physician approves the discharge summary

## Definition of Done

- [ ] Flesch-Kincaid Grade ≤6 verified for 10 diverse test patient scenarios
- [ ] Patient instructions reviewed by a plain-language health literacy expert in UAT

## Dependencies

- EP-004 US-001 (Documentation Agent framework established)
- EP-010 US-002 (Patient portal displays instructions)
