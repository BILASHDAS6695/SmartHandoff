# US-005: Completeness Check Blocks Discharge Until All Required Fields Present

> **Epic:** EP-004 — Documentation Agent | **Sprint:** 2 | **Points:** 3 | **Priority:** Must Have

## User Story

**As a** compliance officer,
**I want** the system to enforce a completeness check on discharge documentation before it can be marked ready for review,
**so that** missing required fields are caught automatically and no discharge summary is approved with critical gaps.

## Acceptance Criteria

- [ ] **AC-1:** Completeness validator checks generated discharge summary JSON against a configurable required-fields checklist (stored in Secret Manager): `primary_diagnosis` (required), `hospital_course` (required), `medications_at_discharge` (required), `follow_up_instructions` (required), `warning_signs` (required)
- [ ] **AC-2:** If any required field is empty or below 10-word minimum: document status stays `DRAFT`; missing fields listed in `Document.completeness_errors` JSON column; SignalR push notifies nurse/physician of incomplete document
- [ ] **AC-3:** Discharge workflow in EP-003 checks for `Document.status=PENDING_REVIEW` (all required docs complete) before allowing encounter status to advance to `DISCHARGED`; if incomplete, discharge is blocked and care coordinator alerted
- [ ] **AC-4:** Completeness rules configurable without code deployment (via Secret Manager JSON config); new required fields take effect on next agent restart
- [ ] **AC-5:** Integration test: generate document with missing `warning_signs` field → verify `Document.status=DRAFT`, completeness error logged, discharge workflow blocked

## Definition of Done

- [ ] Completeness check verified against 10 test document scenarios including edge cases (empty strings, whitespace-only)
- [ ] Discharge blocking tested end-to-end: A03 event → incomplete document → discharge status NOT advanced

## Dependencies

- EP-004 US-001 (Document generation pipeline)
- EP-003 US-001 (Coordinator Agent checks document status before advancing encounter)
