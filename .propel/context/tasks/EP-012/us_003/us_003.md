# US-003: Nightly De-Identified BigQuery Export for Research Analytics

> **Epic:** EP-012 — Analytics & KPI Reporting | **Sprint:** 2 | **Points:** 5 | **Priority:** Should Have

## User Story

**As a** data analyst,
**I want** de-identified encounter data exported nightly to BigQuery,
**so that** I can run longitudinal research queries and build dashboards without accessing any PHI.

## Acceptance Criteria

- [ ] **AC-1:** Nightly Cloud Run job (scheduled via Cloud Scheduler at 02:00 AM) queries `mv_kpi_daily` + encounter table; strips all PHI columns (MRN, first_name, last_name, dob, phone, email); exports de-identified records to BigQuery dataset `smarthandoff_analytics`
- [ ] **AC-2:** De-identification confirmed: BigQuery table schema must NOT contain columns: `mrn`, `first_name`, `last_name`, `dob`, `phone`, `email`; schema validated in CI via `bq show --format=prettyjson` assertion
- [ ] **AC-3:** Exported fields: `encounter_id` (UUID — not traceable to patient without SmartHandoff DB access), `admit_date`, `discharge_date`, `primary_diagnosis_code` (ICD-10 chapter only, not full code), `unit`, `risk_tier`, `los_days`, `doc_time_minutes`, `readmitted_30d` (boolean)
- [ ] **AC-4:** BigQuery dataset access restricted to `analytics@smarthandoff.health` service account; no SmartHandoff DB credentials granted to analytics users
- [ ] **AC-5:** Export job logs: records exported count, export duration, any errors — to Cloud Logging; failure fires P2 alert

## Definition of Done

- [ ] Nightly job verified in staging: synthetic data exported to BigQuery with correct schema
- [ ] PHI audit: BigQuery schema confirmed to contain no PHI field names
- [ ] Data analyst role can query BigQuery without SmartHandoff DB credentials

## Dependencies

- EP-DATA US-004 (mv_kpi_daily materialised view)
- EP-TECH US-001 (Cloud Scheduler, BigQuery provisioned)
