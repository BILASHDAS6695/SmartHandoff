# US-063: Export KPI Reports as CSV and PDF

> **Epic:** EP-012 — Analytics & KPI Reporting | **Sprint:** 2 | **Points:** 3 | **Priority:** Should Have
> **Status:** Draft | **Date:** 2026-07-14

---

## User Story

**As a** hospital manager,
**I want to** export the KPI analytics report as a CSV for data analysis or as a formatted PDF for board presentations,
**So that** I can share performance data with executives and committees who need offline access to the data.

---

## Business Value

KPI data trapped inside a web dashboard is inaccessible for board reporting, committee meetings, and annual reviews. CSV/PDF export enables management to communicate SmartHandoff's ROI to hospital leadership, securing continued investment and operational support.

---

## Acceptance Criteria

### Scenario 1: CSV export downloads all KPI data for date range
**Given** a manager calls `GET /api/v1/analytics/export?format=csv&from={}&to={}`
**When** the export is generated
**Then** a CSV file downloads with column headers and all KPI metrics for each day in the date range; no PHI fields appear in the CSV; the file downloads within 5 seconds for a 1-year date range.

### Scenario 2: PDF export renders a formatted report with KPI sections
**Given** a manager clicks "Export PDF" on the analytics dashboard
**When** `GET /api/v1/analytics/export?format=pdf&from={}&to={}` is called
**Then** a PDF downloads with: hospital name header, date range, KPI summary table, and 5 chart images embedded in the report; the PDF is legible and professionally formatted.

### Scenario 3: CSV has no PHI
**Given** a CSV is downloaded for a 30-day KPI report
**When** all columns are inspected
**Then** the CSV contains only aggregated metrics (averages, counts, rates) and dates/unit names; no patient names, MRNs, encounter IDs, or individually identifiable information.

### Scenario 4: Export is manager-role gated
**Given** a nurse attempts `GET /api/v1/analytics/export?format=csv`
**When** the API is called
**Then** the response is `403 Forbidden`.

---

## Definition of Done

- [ ] `GET /api/v1/analytics/export?format=csv|pdf` FastAPI endpoint
- [ ] CSV: `pandas.DataFrame.to_csv()` with headers; `Content-Disposition: attachment; filename=kpi_report_{date_range}.csv`
- [ ] PDF: `reportlab` library; rendered with hospital name, date range, KPI table, and Chart.js chart PNGs (server-rendered)
- [ ] PDF chart images: pre-rendered server-side using `matplotlib` to match dashboard charts
- [ ] De-identification: same rules as US-062 (no PHI in export)
- [ ] RBAC: manager/admin role only
- [ ] Unit tests: CSV column validation (no PHI), PDF content check
- [ ] Code reviewed and approved

---

## Technical Notes

- `reportlab` `SimpleDocTemplate` with `Table` and `Image` flowables
- Chart PNGs: generate via `matplotlib` on the server using the same KPI query as the dashboard
- CSV streaming response: `StreamingResponse` with `text/csv` media type for large date ranges
- Performance: generate PDF in a background task (`BackgroundTasks.add_task`) for large date ranges; return 202 with download URL
