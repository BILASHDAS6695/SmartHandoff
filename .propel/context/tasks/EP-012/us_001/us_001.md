# US-001: KPI Analytics Dashboard with Date Range and Unit Filters

> **Epic:** EP-012 — Analytics & KPI Reporting | **Sprint:** 2 | **Points:** 5 | **Priority:** Should Have

## User Story

**As a** hospital manager,
**I want** a KPI analytics dashboard with date range and unit filters,
**so that** I can track whether SmartHandoff is achieving its target outcomes and identify underperforming units.

## Acceptance Criteria

- [ ] **AC-1:** Analytics module (`/analytics`) lazy-loaded; accessible only to ADMIN and BED_MANAGER roles; all other roles return 403
- [ ] **AC-2:** 5 KPI chart panels using Chart.js 4.x: (1) Average discharge documentation time (minutes) — line chart trend; (2) 30-day readmission rate (%) — bar chart by month; (3) Medication reconciliation completion rate (%) — gauge chart; (4) Bed utilisation (%) — bar chart by unit; (5) Agent task success rate (%) — donut per agent type
- [ ] **AC-3:** Date range picker (from/to date) + unit dropdown filter; filter changes trigger re-fetch from `GET /api/v1/analytics/kpis?from={}&to={}&unit={}`; charts re-render within 3 seconds
- [ ] **AC-4:** KPI API reads from `mv_kpi_daily` materialised view on Cloud SQL read replica; p95 response <500ms for 90-day date range with unit filter
- [ ] **AC-5:** Drill-down: clicking a KPI chart bar/point shows a paginated table of contributing encounters (anonymised — no PHI, encounter ID only) for root-cause analysis

## Definition of Done

- [ ] 5 KPI charts rendered with synthetic data in staging; filter verified to change chart data
- [ ] p95 API response <500ms for 90-day query under 20 concurrent requests
- [ ] RBAC: NURSE role returns 403 on `/analytics` route

## Dependencies

- EP-DATA US-004 (mv_kpi_daily materialised view)
- EP-009 US-001 (Angular PWA scaffold with analytics module)
- EP-011 US-003 (RBAC for analytics access)
