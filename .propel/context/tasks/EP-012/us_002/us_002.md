# US-002: Export KPI Reports as CSV and PDF

> **Epic:** EP-012 — Analytics & KPI Reporting | **Sprint:** 2 | **Points:** 3 | **Priority:** Should Have

## User Story

**As a** hospital manager,
**I want** to export KPI reports as CSV or PDF,
**so that** I can share performance data with the board, finance, or compliance teams in standard formats.

## Acceptance Criteria

- [ ] **AC-1:** "Export CSV" button on Analytics screen triggers `GET /api/v1/analytics/export?from={}&to={}&unit={}&format=csv`; response headers: `Content-Type: text/csv; charset=utf-8`, `Content-Disposition: attachment; filename="SmartHandoff_KPIs_{date}.csv"`
- [ ] **AC-2:** CSV contains: date, unit, avg_discharge_doc_time_minutes, readmission_rate_pct, med_reconciliation_completion_rate_pct, bed_utilisation_pct, agent_success_rate_pct — zero PHI fields (no MRN, name, DOB)
- [ ] **AC-3:** "Export PDF" button triggers `GET /api/v1/analytics/export?format=pdf`; server-side PDF generated using `reportlab`; includes: SmartHandoff logo, report date range, hospital name, all 5 KPI charts rendered as embedded PNG images, data table
- [ ] **AC-4:** CSV and PDF exports contain no PHI — confirmed by automated test parsing export and checking for PHI field names (MRN, first_name, last_name, dob, phone, email)
- [ ] **AC-5:** Export download time <10 seconds for 90-day date range

## Definition of Done

- [ ] CSV and PDF exports tested with 90-day synthetic data in staging
- [ ] PHI audit: automated parser confirms no PHI field names in export content

## Dependencies

- EP-012 US-001 (Analytics KPI API)
- EP-011 US-003 (RBAC — analytics-only roles for export)
