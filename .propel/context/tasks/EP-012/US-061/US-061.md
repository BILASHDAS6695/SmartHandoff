# US-061: Build KPI Analytics Dashboard with Chart.js Visualisations

> **Epic:** EP-012 — Analytics & KPI Reporting | **Sprint:** 2 | **Points:** 5 | **Priority:** Should Have
> **Status:** Draft | **Date:** 2026-07-14

---

## User Story

**As a** hospital manager,
**I want to** view an analytics dashboard with five KPI charts (discharge documentation time, 30-day readmission rate, medication reconciliation completion rate, bed utilisation, agent task success rate) filterable by date range and unit,
**So that** I can track whether SmartHandoff is delivering on its promised business outcomes and identify operational areas needing improvement.

---

## Business Value

Analytics visibility is how hospital management validates ROI (BO-01 through BO-05 targets). Without measurable KPI dashboards, the value of SmartHandoff is invisible to leadership. Real-time KPI access enables rapid operational adjustments that keep the system continuously improving rather than stagnating after initial deployment.

---

## Acceptance Criteria

### Scenario 1: KPI dashboard renders within 3 seconds with date range filter
**Given** a manager navigates to `/analytics`
**When** the page loads with default 30-day date range
**Then** all 5 KPI charts render within 3 seconds; each chart shows correct aggregated values from `mv_kpi_daily`; date range picker is pre-set to the last 30 days.

### Scenario 2: Date range filter updates all charts simultaneously
**Given** the manager changes the date range to "Last 7 days"
**When** the filter is applied
**Then** all 5 charts update within 2 seconds to reflect the new date range; chart axes auto-scale to the filtered data.

### Scenario 3: Analytics data contains no PHI
**Given** the KPI API response is inspected
**When** `GET /api/v1/analytics/kpis?from={}&to={}&unit={}` is called
**Then** the response contains only aggregated metrics (counts, averages, percentages) — no patient names, MRNs, DOBs, or any individually identifiable information.

### Scenario 4: Manager role can access analytics; Nurse cannot
**Given** a nurse JWT with `role=NURSE` calls `GET /api/v1/analytics/kpis`
**When** the request is processed
**Then** the response is `403 Forbidden`; analytics is only accessible to `MANAGER` and `ADMIN` roles.

## Definition of Done

- [ ] `AnalyticsComponent` Angular lazy-loaded module with 5 Chart.js 4.x charts
- [ ] Chart types: discharge_time → Line, readmission_rate → Bar, med_recon_rate → Gauge, bed_utilisation → Doughnut, agent_success_rate → Stacked Bar
- [ ] `GET /api/v1/analytics/kpis` FastAPI endpoint: queries `mv_kpi_daily` via read replica; de-identified aggregated metrics only
- [ ] Date range filter: `MatDateRangePicker` component; URL query params `?from=&to=&unit=`
- [ ] Unit filter dropdown populated from `app_user.units` (manager's accessible units)
- [ ] RBAC: manager/admin role only (US-057 RBAC dependency)
- [ ] Unit tests: API endpoint de-identification, RBAC enforcement
- [ ] Code reviewed and approved

---

## Technical Notes

- `mv_kpi_daily` columns: `date`, `unit`, `avg_discharge_doc_time_min`, `readmission_rate_30d`, `med_recon_completion_rate`, `bed_utilisation_pct`, `agent_task_success_rate`
- Chart.js `ng2-charts` Angular wrapper for Chart.js 4.x
- API response: only aggregated values — no encounter IDs, patient identifiers, or names
- Date arithmetic in SQL: `EXTRACT(EPOCH FROM avg(completed_at - created_at))` for time calculations
