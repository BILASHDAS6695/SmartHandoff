# US-003: Archive Raw HL7 Messages to Cloud Storage Before ACK

> **Epic:** EP-001 — HL7 ADT Event Ingestion | **Sprint:** 1 | **Points:** 2 | **Priority:** Must Have

## User Story

**As a** compliance officer,
**I want** every raw HL7 message written to a HIPAA-compliant Cloud Storage bucket before the ACK is sent to the EHR,
**so that** I can replay, audit, or investigate any care transition event for the full 7-year retention period.

## Acceptance Criteria

- [ ] **AC-1:** Raw HL7 message bytes written to `gs://smarthandoff-hl7-archive/{yyyy}/{MM}/{dd}/{MSH-10-message-id}.hl7` before ACK is returned to EHR
- [ ] **AC-2:** Cloud Storage bucket configured with: CMEK encryption, uniform bucket-level access, object versioning enabled, retention policy 7 years (Object Lock)
- [ ] **AC-3:** If Cloud Storage write fails (timeout, auth error): message still ACK'd to EHR; Cloud Monitoring P2 alert fired; structured error log emitted with message ID for manual archival
- [ ] **AC-4:** `adt_event.source_message_id` (MSH-10) stored in DB and matches the filename in Cloud Storage — enabling lookup by message ID
- [ ] **AC-5:** Integration test: send 100 HL7 messages → verify 100 `.hl7` files in Cloud Storage with correct path structure and content

## Definition of Done

- [ ] Cloud Storage write verified in staging environment with correct CMEK and retention policy
- [ ] Failure path tested: Cloud Storage unreachable → ACK still sent → alert fires

## Dependencies

- EP-001 US-001 (MLLP listener)
- EP-TECH US-001 (Cloud Storage HL7 archive bucket)
