# US-001: Build MLLP TCP Listener with ACK/NACK within 200ms

> **Epic:** EP-001 — HL7 ADT Event Ingestion | **Sprint:** 1 | **Points:** 5 | **Priority:** Must Have

## User Story

**As a** backend engineer,
**I want** an asyncio-based MLLP TCP listener on port 2575 that acknowledges valid HL7 messages within 200ms and NACKs invalid ones,
**so that** the EHR receives immediate confirmation of message receipt and parse failures are surfaced clearly.

## Acceptance Criteria

- [ ] **AC-1:** MLLP listener binds TCP port 2575; accepts up to 50 concurrent connections with TCP keep-alive enabled and idle timeout of 300 seconds
- [ ] **AC-2:** Valid `ADT^A01` message → raw HL7 archived to Cloud Storage → `ADTEvent` persisted to DB → Pub/Sub published → ACK (`AA`) returned; total elapsed time ≤200ms measured from TCP receive to ACK send
- [ ] **AC-3:** Invalid message (missing MSH segment, unknown event type, malformed MLLP framing) → NACK (`AE`) returned with error description; message archived to Cloud Storage for replay; structured error log emitted (no PHI in log)
- [ ] **AC-4:** Load test: 50 concurrent MLLP connections each sending 100 messages/minute → all messages processed, zero message loss, zero dropped connections
- [ ] **AC-5:** `GET /health` returns 200 when listener is ready; `GET /ready` returns 200 only after successful Pub/Sub and DB connectivity checks

## Definition of Done

- [ ] Integration test sends real HL7 v2.x ADT^A01 fixture and verifies ACK within 200ms
- [ ] Load test confirms no message loss at 50 concurrent connections
- [ ] Deployed as Cloud Run service with min-instances=1

## Dependencies

- EP-TECH US-001 (Cloud Run, Cloud Storage, Pub/Sub provisioned)
- EP-DATA US-001 (adt_event table available)
