# US-002: Parse All 8 HL7 ADT Event Types to ADTEvent Domain Model

> **Epic:** EP-001 — HL7 ADT Event Ingestion | **Sprint:** 1 | **Points:** 3 | **Priority:** Must Have

## User Story

**As a** backend engineer,
**I want** the HL7 parser to extract PID, PV1, EVN, and DG1 segments and map all 8 ADT event types to the `ADTEvent` domain model,
**so that** every care transition event type is correctly classified and routable to the appropriate agent workflow.

## Acceptance Criteria

- [ ] **AC-1:** Parser handles all 8 event types: A01 (Admit), A02 (Transfer), A03 (Discharge), A04 (Register), A08 (Update), A11 (Cancel Admit), A12 (Cancel Transfer), A13 (Cancel Discharge)
- [ ] **AC-2:** From PID segment: `patient_id` (MRN from PID-3), `patient_name` (PID-5), `dob` (PID-7), `gender` (PID-8), `phone` (PID-13), `preferred_language` (PID-15) extracted correctly
- [ ] **AC-3:** From PV1 segment: `unit` (PV1-3), `attending_md` (PV1-7), `admit_date` (PV1-44), `discharge_date` (PV1-45) extracted
- [ ] **AC-4:** `ADTEvent` record populated with: `event_type`, `event_time` (EVN-2), `source_system` (MSH-3), `source_message_id` (MSH-10)
- [ ] **AC-5:** 20 HL7 test fixture files (one per event subtype + edge cases) all parse without error; unit tests verify each mapped field
- [ ] **AC-6:** Unknown event type beyond A01–A13 returns `ParseError` → NACK (AE) with unknown-event-type error code

## Definition of Done

- [ ] All 20 test fixtures pass; parser coverage ≥90% for HL7 parsing module
- [ ] Code review approved by a backend engineer familiar with HL7 v2.x

## Dependencies

- EP-001 US-001 (MLLP listener exists)
- EP-DATA US-001 (ADTEvent ORM model defined)
